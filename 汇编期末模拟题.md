## 一、单项选择题

1. 当前课程编程题默认使用的汇编环境最符合哪一项？
   A. MASM + 16 位 DOS + AT&T 语法  
   B. GAS + x86-64 + Intel 语法  
   C. ARMv8 + A64 + Windows API  
   D. NASM + 8086 实模式

   **答案：B**

2. 从 `.s` 源文件生成可执行文件，一般至少经历哪两个关键阶段？
   A. 编译和解释  
   B. 汇编和链接  
   C. 压缩和装载  
   D. 反汇编和调试

   **答案：B**

3. 下列哪个文件通常表示目标文件？
   A. `.s`  
   B. `.o`  
   C. `.md`  
   D. `.png`

   **答案：B**

4. GAS Intel 语法源文件开头常写哪条伪指令？
   A. `.model small`  
   B. `.intel_syntax noprefix`  
   C. `.syntax divided`  
   D. `.code16`

   **答案：B**

5. 在 GAS 中导出 `main` 符号常用：
   A. `.extern main`  
   B. `.public main`  
   **C. `.globl main`**  
   D. `.entry main`

   **答案：C**

6. `.asciz "abc"` 在内存中占多少字节？
   A. 3  
   B. 4  
   C. 6  
   D. 8

   **答案：B**

7. `.long 0x12345678` 在 x86-64 小端机器中从低地址到高地址存放为：
   A. `12 34 56 78`  
   B. `78 56 34 12`  
   C. `34 12 78 56`  
   D. `56 78 12 34`

   **答案：B**

8. 主存储器通常采用：
   A. 位编址  
   B. 字节编址  
   C. 字编址且每字固定 64 位  
   D. 只能按双字编址

   **答案：B**

9. 8 位有符号补码能表示的范围是：
   A. `0 ~ 255`  
   B. `-127 ~ 128`  
   C. `-128 ~ 127`  
   D. `-255 ~ 255`

   **答案：C**

10. `.equ N, 10` 的主要含义是：
    A. 分配 10 字节空间  
    B. 定义符号常量 `N`  
    C. 把 `N` 放入寄存器  
    D. 定义字符串

   **答案：B**

11. `.space 8, 0xff` 的含义最接近：
    A. 分配 8 字节并用 `0xff` 填充  
    B. 定义 8 个字符串  
    C. 把栈空间扩大 8 字节  
    D. 定义 8 个 `.quad`

   **答案：A**

12. x86-64 常用的 64 位通用寄存器不包括：
    A. `rax`  
    B. `rdi`  
    C. `rip`  
    D. `r15`

   **答案：C**

13. 在 64 位模式中，写入 `eax` 通常会：
    A. 只修改 `rax` 低 8 位  
    B. 修改 `rax` 低 32 位并把高 32 位清 0

14. 下列哪组 8 位寄存器通常不能在同一条指令中与 REX 前缀新增低 8 位寄存器混用？
    A. `al/bl/cl/dl`  
    B. `ah/bh/ch/dh`  
    C. `r8b/r9b/r10b/r11b`

15. 访问全局变量 `x` 的课件主线写法优先使用：

    B. `[bp+si]`  
    C. `ds:x`

16. 下列寻址表达式中，比例因子合法的是：

30. 有符号 32 位除法 `idiv ebx` 前，通常用哪条指令把 `EAX` 符号扩展到 `EDX:EAX`？
    A. `cqo`

## 二、判断题

1. GAS Intel 语法中，当前课程常用 `.intel_syntax noprefix`。
   **答案：对**
2. `.ascii "abc"` 会自动在末尾补 `\0`。
   **答案：错，`.ascii` 不自动补 0，`.asciz` / `.string` 补 0。**
3. x86-64 小端方式中，低位字节放低地址。
   **答案：对**
4. `cmp` 会保存减法结果到目的操作数。
   **答案：错，`cmp` 只设置标志，不保存结果。**
5. `test` 常用于判断某寄存器是否为 0 或某一位是否为 1。
   **答案：对**
6. `ja/jb` 通常用于无符号比较，`jg/jl` 通常用于有符号比较。
   **答案：对**
7. `CF` 和 `OF` 含义相同，可以随便互换。
   **答案：错，`CF` 看无符号进位/借位，`OF` 看有符号溢出。**
8. `loop` 指令会修改 `ZF`，所以会破坏前一条 `cmp` 的结果。
   **答案：错，`loop` 不影响标志位。**
9. 32 位写入 `eax` 会把 `rax` 高 32 位清 0。
   **答案：对**
10. `div/idiv` 的被除数寄存器由操作数宽度隐含规定。
   **答案：对**
11. 32 位 `idiv` 前常用 `cdq` 准备 `EDX:EAX`。
   **答案：对**
12. `mul r/m32` 是无符号乘法。
   **答案：对**
13. `add` 后接 `adc` 可用于多精度加法。
   **答案：对**
14. `sub` 后接 `sbb` 可用于多精度减法。
   **答案：对**
15. `push rax` 在 64 位下通常使 `rsp` 减 8。
   **答案：对**
16. `call` 会保存返回地址，`jmp` 不会。
   **答案：对**
17. `ret n` 可以理解为普通 `ret` 后再额外调整栈指针。
   **答案：对**
18. Linux x86-64 下第一个 `int` 参数常看 `edi`。
   **答案：对**
19. Win64 下调用 API 常要注意 32 字节影子空间。
   **答案：对**
20. 宏调用需要 `call`，宏定义内部必须写 `ret`。
   **答案：错，宏是汇编时展开，不用 `call/ret`。**
21. `.globl func` 可以把本模块定义的 `func` 暴露给链接器。
   **答案：对**
22. `.extern` 在 GAS 中表示外部符号，很多情况下也可不显式写。
   **答案：对**
23. 基本 `asm("...")` 适合可靠地直接读写 C 变量。
   **答案：错，基本 asm 不适合可靠直接和 C 变量交互，应用扩展 asm。**
24. 扩展 asm 中 `"memory"` 表示这段汇编可能读写内存。
   **答案：对**
25. x86 属于 CISC，ARM/AArch64 属于 RISC。
   **答案：对**
26. 8086 实模式一个段最大通常为 64KB。
   **答案：对**
27. 实模式中一个物理地址只能有一个逻辑地址表示。
   **答案：错，一个物理地址可以有多个逻辑地址表示。**
28. AArch64 中 `EL1` 通常对应操作系统内核。
   **答案：对**
29. AArch64 中 `SPSR_ELn` 用于保存异常发生时的 `PSTATE`。
   **答案：对**
30. `MessageBoxA` 属于 `USER32.DLL`。
   **答案：对**

## 三、填空与简答题

1. 写出当前课程最常用的 GAS 程序基本框架，至少包含语法声明、代码段、导出 `main`、`main` 标签和返回。

   **答案：**

示例：
```asm
.intel_syntax noprefix
.text
.globl main
main:
    mov eax, 0
    ret
```

2. 写出 `.byte`、`.word`、`.long`、`.quad` 分别对应的字节数。

   **答案：**

`.byte` 1 字节，`.word` 2 字节，`.long` 4 字节，`.quad` 8 字节。

3. `.asciz "hi"` 在内存中依次存放哪些字节？占几个字节？

   **答案：**

`68 69 00`，共 3 字节。

4. 写出 n 位无符号数和 n 位有符号补码的表示范围。

   **答案：**

n 位无符号：`0 ~ 2^n-1`；n 位有符号补码：`-2^(n-1) ~ 2^(n-1)-1`。

5. 把 8 位数 `0xB6` 解释为有符号补码，十进制是多少？

   **答案：**

`0xB6 = 182`，按 8 位有符号为 `182 - 256 = -74`。

6. `.long 0x9ABCDEF0` 在小端机器中从低地址到高地址如何排列？

   **答案：**

`F0 DE BC 9A`。

7. 已知内存从 `array` 开始为 `01 00 02 00`，执行 `mov eax, dword ptr array[rip]` 后 `EAX` 是多少？

   **答案：**

`EAX = 0x00020001`。

8. 说明 `.equ` 与 `.set` 的主要区别。

   **答案：**

`.equ` 定义后不可改；`.set` 可以重新赋值。

9. 说明 `label[rip]` 这种写法在当前课程中的作用。

   **答案：**

用 RIP 相对寻址访问全局/静态数据，适合当前 x86-64 课程口径。

10. 写出下列地址表达式 `[rdi+rax*4+8]` 中的基址、变址、比例因子、位移。

   **答案：**

基址 `rdi`，变址 `rax`，比例因子 4，位移 8。

11. 指出下面指令的问题并改正：`mov [rdi], [rsi]`。

   **答案：**

x86 通常不允许内存到内存直接传送。可改为：
```asm
mov eax, dword ptr [rsi]
mov dword ptr [rdi], eax
```

12. 指出下面指令的问题并改正：`mov 10, eax`。

   **答案：**

立即数不能作目的操作数。可改为 `mov eax, 10` 或 `add eax, 10`，看题意。

13. `xor eax, eax` 执行后，`EAX`、`ZF`、`SF`、`CF`、`OF` 通常是什么？

   **答案：**

`EAX=0`，`ZF=1`，`SF=0`，`CF=0`，`OF=0`。

14. 8 位计算 `0x7f + 0x01` 的结果是多少？`CF`、`OF`、`SF`、`ZF` 分别是多少？

   **答案：**

8 位结果 `0x80`；`CF=0`，`OF=1`，`SF=1`，`ZF=0`。

15. 8 位计算 `0x00 - 0x01` 的结果是多少？`CF`、`OF`、`SF`、`ZF` 分别是多少？

   **答案：**

8 位结果 `0xff`；`CF=1`，`OF=0`，`SF=1`，`ZF=0`。

16. 简述 `CF` 与 `OF` 的区别。

   **答案：**

`CF` 是无符号进位/借位；`OF` 是有符号溢出。

17. 简述 `cmp eax, ebx` 后 `je`、`ja`、`jg` 分别按什么条件判断。

   **答案：**

`je` 看 `ZF=1`；`ja` 按无符号 `eax > ebx`；`jg` 按有符号 `eax > ebx`。

18. 写出无符号 `>`、`>=`、`<`、`<=` 常用跳转指令。

   **答案：**

无符号：`ja`、`jae/jnb`、`jb/jc`、`jbe`。

19. 写出有符号 `>`、`>=`、`<`、`<=` 常用跳转指令。

   **答案：**

有符号：`jg/jnle`、`jge/jnl`、`jl/jnge`、`jle/jng`。

20. 说明 `test al, 1` 后用什么跳转判断 AL 是否为奇数。

   **答案：**

`test al, 1` 后用 `jnz` 判断奇数。

21. 说明 `bt eax, 5` 后用什么跳转判断第 5 位是否为 1。

   **答案：**

`bt eax, 5` 后用 `jc` 判断第 5 位为 1。

22. 写出 `loop L` 的近似伪代码，并指出它是否影响标志位。

   **答案：**

近似：
```text
ecx = ecx - 1
if ecx != 0 jump L
```
`loop` 不影响标志位。

23. 为什么进入 `loop` 循环前常先用 `jecxz done`？

   **答案：**

避免 `ECX=0` 时执行 `loop` 先减成 `0xffffffff`，造成超大循环。

24. 说明 `shl`、`shr`、`sar` 的区别。

   **答案：**

`shl` 左移低位补 0；`shr` 逻辑右移高位补 0；`sar` 算术右移高位补符号位。

25. 写出 32 位有符号除法中被除数、除数、商、余数所在位置。

   **答案：**

被除数 `EDX:EAX`，除数 r/m32，商 `EAX`，余数 `EDX`。

26. 写出 64 位有符号除法前常用的符号扩展指令。

   **答案：**

`cqo`，把 `RAX` 符号扩展到 `RDX:RAX`。

27. 写出 128 位加法中低 64 位和高 64 位分别使用的指令。

   **答案：**

低 64 位 `add`，高 64 位 `adc`。

28. 写出 `push rax` 和 `pop rdx` 对 `rsp` 的影响。

   **答案：**

`push rax`：`rsp -= 8`；`pop rdx`：`rdx=[rsp]` 后 `rsp += 8`。

29. 简述 `call` 与 `ret` 的配合过程。

   **答案：**

`call` 把返回地址压栈并跳到子程序；子程序 `ret` 从栈顶取返回地址回到调用点下一条指令。

30. 写出 Linux x86-64 ABI 前 6 个整数/指针参数寄存器。

   **答案：**

`rdi, rsi, rdx, rcx, r8, r9`。

31. 写出 Win64 ABI 前 4 个整数/指针参数寄存器。

   **答案：**

`rcx, rdx, r8, r9`。

32. 说明“入口参数”和“出口参数”的区别。

   **答案：**

入口参数是主程序传给子程序的数据；出口参数是子程序返回给主程序的数据。

33. 列出课件要求掌握的三种参数传递方式。

   **答案：**

寄存器传参、共享变量传参、堆栈传参。

34. `.include` 与多模块链接的区别是什么？

   **答案：**

`.include` 是源文本包含；多模块链接是各文件分别生成 `.o` 后由链接器合成可执行文件。

35. 宏与子程序至少写出 3 个区别。

   **答案：**

宏汇编时展开、子程序运行时调用；宏不用 `call/ret`、子程序通常用；宏每次展开可能增加代码体积、子程序只有一份代码；宏无调用返回开销、子程序有。

```c
    : 输出操作数

```

   **答案：**

38. 写出 ARM 异常等级 `EL0`、`EL1`、`EL2`、`EL3` 的常见含义。

   **答案：**

## 四、代码阅读与分析题

### 1. 小端与取数

```asm
.data
array:
    .word 1, 2, 3, 4

.text
    mov eax, dword ptr array[rip]
    mov bx,  word ptr array+4[rip]
```

回答：

1. `array` 开始的 8 个字节从低地址到高地址是什么？
2. `EAX` 的值是多少？
3. `BX` 的值是多少？

**答案：**

1. `.word 1,2,3,4` 从低地址到高地址：`01 00 02 00 03 00 04 00`。
2. `EAX = 0x00020001`。
3. `BX = 0x0003`。

### 2. 标志位

假设只看 8 位运算：

```asm
mov al, 0xff
add al, 1
```

回答 `AL`、`CF`、`OF`、`SF`、`ZF`、`PF` 的值。

**答案：**

`0xff + 1 = 0x100`，截断到 8 位后 `AL=0x00`。
`CF=1`，`OF=0`，`SF=0`，`ZF=1`，`PF=1`。

### 3. 有符号与无符号跳转

```asm
mov al, 0xff
cmp al, 1
ja  L1
jl  L2
```

回答：

1. `ja L1` 是否会跳？为什么？
2. `jl L2` 是否会跳？为什么？

**答案：**

1. `ja L1` 会跳；`0xff` 按无符号是 255，`255 > 1`。
2. 若没有先跳走，`jl L2` 也满足；`0xff` 按 8 位有符号是 -1，`-1 < 1`。实际顺序执行时会先跳到 `L1`，后面的 `jl` 是否执行取决于 `L1` 的位置和流程。

### 4. loop 与标志

```asm
mov ecx, 3
mov eax, 5
cmp eax, 5
L:
    nop
    loop L
je equal
```

回答：`je equal` 看的是哪条指令设置的 `ZF`？为什么？

**答案：**

`je equal` 看的是 `cmp eax, 5` 设置的 `ZF`。因为 `loop` 不影响标志位，中间的 `nop` 也不影响标志位。

### 5. 乘除法

```asm
mov eax, -17
mov ebx, 5
cdq
idiv ebx
```

回答：执行后 `EAX` 和 `EDX` 分别是什么？说明商和余数的符号规则。

**答案：**

`-17 / 5` 商为 `-3`，余数为 `-2`。
所以 `EAX=-3`，`EDX=-2`。`idiv` 的商向 0 截断，余数与被除数同号。

### 6. 128 位加法

设：

```text
A = A_hi:A_lo
B = B_hi:B_lo
```

补全下面代码，实现 `A = A + B`，其中 `A_lo` 在 `rax`，`A_hi` 在 `rdx`，`B_lo` 在 `rbx`，`B_hi` 在 `rcx`。

```asm
____ rax, rbx
____ rdx, rcx
```

**答案：**

```asm
add rax, rbx
adc rdx, rcx
```

### 7. 栈与返回地址

```asm
main:
    mov eax, 1
    call sub
    add eax, 2
    ret

sub:
    add eax, 3
    ret
```

回答：

1. `call sub` 会把哪个地址压栈？
2. `sub` 返回后下一条执行哪条指令？
3. 如果忽略 `main` 最后的返回，`eax` 最终是多少？

**答案：**

1. 压入 `call sub` 后面那条指令的地址，也就是 `add eax, 2` 的地址。
2. 返回后执行 `add eax, 2`。
3. `eax = 1 + 3 + 2 = 6`。

### 9. Win64 参数识别


## 五、改错题

指出错误并给出一种正确写法。

1. 
```asm
mov dword ptr [rdi], dword ptr [rsi]
```

**答案：**

内存到内存非法，改：
```asm
mov eax, dword ptr [rsi]
mov dword ptr [rdi], eax
```

2. 
```asm
add 1, eax
```

**答案：**

立即数不能作目的操作数，改：
```asm
add eax, 1
```

3. 
```asm
mov rax, dword ptr [rdi]
```

**答案：**

位宽不匹配。若要零扩展读 32 位到 64 位，可写：
```asm
mov eax, dword ptr [rdi]
```
写 `eax` 会清 `rax` 高 32 位。

4. 
```asm
mov ah, r8b
```

**答案：**

`ah/bh/ch/dh` 不能与需要 REX 前缀的新 8 位寄存器混用。可改用低 8 位寄存器，例如：
```asm
mov al, r8b
```
或调整寄存器分配。

5. 
```asm
idiv ebx
```
已知要计算 32 位有符号 `EAX / EBX`，但前面没有准备 `EDX:EAX`。

**答案：**

应先准备符号扩展：
```asm
cdq
idiv ebx
```

6. 
```asm
div ebx
```
已知要计算 32 位无符号 `EAX / EBX`，但 `EDX` 中有旧值。

**答案：**

应先清高 32 位：
```asm
xor edx, edx
div ebx
```

7. 
```asm
cmp eax, ebx
ja signed_greater
```
题意是判断有符号 `eax > ebx`。

**答案：**

有符号大于应使用：
```asm
cmp eax, ebx
jg signed_greater
```

8. 
```asm
.globl add
add:
    mov eax, edi
    add eax, esi
    push rax
    ret
```

**答案：**

`push rax` 破坏返回地址位置，`ret` 会把刚压入的 `rax` 当返回地址。应删去 `push rax`：
```asm
.globl add
add:
    mov eax, edi
    add eax, esi
    ret
```

9. 
```asm
.macro M a
    call \a
    ret
.endm
```
题意是定义一个普通代码替换宏，不是调用子程序。

**答案：**

普通宏不需要 `call/ret`，应写成要展开的指令体。例如：
```asm
.macro M a
    mov eax, \a
.endm
```
具体内容看题意。

10. 
```asm
main:
    call func
func:
    ret
```
如果希望 `func` 返回后继续执行 `main` 后续代码，这段有什么结构风险？如何改？

**答案：**

`call func` 后面紧跟 `func:`，若想返回后执行主程序后续代码，应把后续代码放在 `call` 后，并用跳转越过子程序定义，或把子程序放到 `main` 的 `ret` 后。例如：
```asm
main:
    call func
    mov eax, 0
    ret

func:
    ret
```

16. `CF` 是无符号进位/借位；`OF` 是有符号溢出。
17. `je` 看 `ZF=1`；`ja` 按无符号 `eax > ebx`；`jg` 按有符号 `eax > ebx`。
18. 无符号：`ja`、`jae/jnb`、`jb/jc`、`jbe`。
19. 有符号：`jg/jnle`、`jge/jnl`、`jl/jnge`、`jle/jng`。
20. `test al, 1` 后用 `jnz` 判断奇数。
21. `bt eax, 5` 后用 `jc` 判断第 5 位为 1。
22. 近似：
```text
ecx = ecx - 1
if ecx != 0 jump L
```
`loop` 不影响标志位。
23. 避免 `ECX=0` 时执行 `loop` 先减成 `0xffffffff`，造成超大循环。
24. `shl` 左移低位补 0；`shr` 逻辑右移高位补 0；`sar` 算术右移高位补符号位。
25. 被除数 `EDX:EAX`，除数 r/m32，商 `EAX`，余数 `EDX`。
26. `cqo`，把 `RAX` 符号扩展到 `RDX:RAX`。
27. 低 64 位 `add`，高 64 位 `adc`。
28. `push rax`：`rsp -= 8`；`pop rdx`：`rdx=[rsp]` 后 `rsp += 8`。
29. `call` 把返回地址压栈并跳到子程序；子程序 `ret` 从栈顶取返回地址回到调用点下一条指令。
30. `rdi, rsi, rdx, rcx, r8, r9`。
31. `rcx, rdx, r8, r9`。
32. 入口参数是主程序传给子程序的数据；出口参数是子程序返回给主程序的数据。
33. 寄存器传参、共享变量传参、堆栈传参。
34. `.include` 是源文本包含；多模块链接是各文件分别生成 `.o` 后由链接器合成可执行文件。
35. 宏汇编时展开、子程序运行时调用；宏不用 `call/ret`、子程序通常用；宏每次展开可能增加代码体积、子程序只有一份代码；宏无调用返回开销、子程序有。
36. 
```c
asm volatile(
    "汇编模板"
    : 输出操作数
    : 输入操作数
    : 破坏列表
);
```
37. `0x1400 * 16 + 0x0100 = 0x14100`。
38. `EL0` 应用程序，`EL1` OS 内核，`EL2` Hypervisor，`EL3` 安全监控/TrustZone。
39. `ELR_ELn` 保存异常发生前的 PC；`SPSR_ELn` 保存异常发生时的 `PSTATE`。
40. Win64 ABI 要求调用者为前 4 个寄存器参数预留 32 字节影子空间，并保持栈 16 字节对齐，调用 API 时尤其容易考。

## 四、代码阅读与分析题答案

### 1. 小端与取数

1. `.word 1,2,3,4` 从低地址到高地址：`01 00 02 00 03 00 04 00`。
2. `EAX = 0x00020001`。
3. `BX = 0x0003`。

### 2. 标志位

`0xff + 1 = 0x100`，截断到 8 位后 `AL=0x00`。  
`CF=1`，`OF=0`，`SF=0`，`ZF=1`，`PF=1`。

### 3. 有符号与无符号跳转

1. `ja L1` 会跳；`0xff` 按无符号是 255，`255 > 1`。
2. 若没有先跳走，`jl L2` 也满足；`0xff` 按 8 位有符号是 -1，`-1 < 1`。实际顺序执行时会先跳到 `L1`，后面的 `jl` 是否执行取决于 `L1` 的位置和流程。

### 4. loop 与标志

`je equal` 看的是 `cmp eax, 5` 设置的 `ZF`。因为 `loop` 不影响标志位，中间的 `nop` 也不影响标志位。

### 5. 乘除法

`-17 / 5` 商为 `-3`，余数为 `-2`。  
所以 `EAX=-3`，`EDX=-2`。`idiv` 的商向 0 截断，余数与被除数同号。

### 6. 128 位加法

```asm
add rax, rbx
adc rdx, rcx
```

### 7. 栈与返回地址

1. 压入 `call sub` 后面那条指令的地址，也就是 `add eax, 2` 的地址。
2. 返回后执行 `add eax, 2`。
3. `eax = 1 + 3 + 2 = 6`。

### 8. Linux ABI 子程序

```c
int f(int a, int b, int c)
{
    return a * b + c;
}
```

### 9. Win64 参数识别

1. 更像 Win64 ABI。
2. 前四个参数：`rcx, rdx, r8, r9`。
3. 与 Win64 的 32 字节影子空间和 16 字节栈对齐有关；这里 `40 = 32 + 8` 常用于配合调用前对齐。

### 10. 宏展开

展开为：

```asm
mov eax, ebx
xor eax, ecx
```

宏是汇编时文本/代码展开，不压返回地址，不用 `ret`；子程序是运行时通过 `call/ret` 跳转执行。

## 五、改错题答案

1. 内存到内存非法，改：
```asm
mov eax, dword ptr [rsi]
mov dword ptr [rdi], eax
```

2. 立即数不能作目的操作数，改：
```asm
add eax, 1
```

3. 位宽不匹配。若要零扩展读 32 位到 64 位，可写：
```asm
mov eax, dword ptr [rdi]
```
写 `eax` 会清 `rax` 高 32 位。

4. `ah/bh/ch/dh` 不能与需要 REX 前缀的新 8 位寄存器混用。可改用低 8 位寄存器，例如：
```asm
mov al, r8b
```
或调整寄存器分配。

5. 应先准备符号扩展：
```asm
cdq
idiv ebx
```

6. 应先清高 32 位：
```asm
xor edx, edx
div ebx
```

7. 有符号大于应使用：
```asm
cmp eax, ebx
jg signed_greater
```

8. `push rax` 破坏返回地址位置，`ret` 会把刚压入的 `rax` 当返回地址。应删去 `push rax`：
```asm
.globl add
add:
    mov eax, edi
    add eax, esi
    ret
```

9. 普通宏不需要 `call/ret`，应写成要展开的指令体。例如：
```asm
.macro M a
    mov eax, \a
.endm
```
具体内容看题意。

10. `call func` 后面紧跟 `func:`，若想返回后执行主程序后续代码，应把后续代码放在 `call` 后，并用跳转越过子程序定义，或把子程序放到 `main` 的 `ret` 后。例如：
```asm
main:
    call func
    mov eax, 0
    ret

func:
    ret
```
