# 转移指令

**同一串二进制，既可以当无符号数看，也可以当有符号数看。CPU 的 cmp 不知道你想怎么看，真正决定“怎么看”的是后面的跳转指令。**

## 无符号比较用ja/jae/jb/jbe（a等于above，b等于below）（**ab无lg有**

因为没有符号，所以看CF和ZF

cmp a, b就是a-b,如果无符号下a<b，那么要借位所以cf = 1

所以jb label（如果无符号小于，则跳转label)

| 指令    | 含义      | 类似 C 语言 |
| ----- | ------- | ------- |
| `ja`  | 无符号大于   | `>`     |
| `jae` | 无符号大于等于 | `>=`    |
| `jb`  | 无符号小于   | `<`     |
| `jbe` | 无符号小于等于 | `<=`    |

## 有符号：jg/jge/jl/jle

（g是greater，l是less）

要看符号位；jge等价于jnl（大于等于等价于不小于）

## jc：CF为1跳转

## jns：jump if not sign：SF为0跳

# 指令指针：x86-64里面是RIP：当前或者下一步要执行的指令地址

所以改变指令指针会改变程序流程

# 短转移远转移近转移

## 短转移short jump

跳转距离比较短
使用 **8 位有符号位移**
范围大约是 -128 ~ +127 字节

**EB xx一般就是短**

## 近转移 near jump

同一个代码段内跳转，只改变RIP不改变cs（代码段寄存器）

jmp label如果label再同一个代码段内就是

## 远转移 far jump

跳到另一个代码段
既改变 CS，也改变 IP/EIP


# JMP指令属于程序控制类指令

默认按顺序，但是如果执行了jmp labelcpu执行的位置就会改变

所以顺序类不应该有jmp或者jz之类的

## 方式

### 直接转移：jmp again，转移到again

### 间接：例如jmp rbx，跳到RBX里面保存的地址

# 标号（比如again）

是代码标号，表示某条指令所在的位置带有

段内 / 段间属性（判断是段内转移还是段间，段内只改变RIP，段间还要改CS）
near / far 属性（near表段内，far表示段间）
地址信息（逻辑地址）

但不是普通数据变量，一般不说是byte类型或者dword类型
