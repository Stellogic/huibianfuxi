# c程序怎么变成可执行

C源程序 .c
   ↓ 预处理  gcc -E
预处理文件 .i        仍然是文本文件
   ↓ 编译    gcc -S
汇编文件 .s          仍然是文本文件
   ↓ 汇编    gcc -c 或 as
目标文件 .o          二进制文件，但还不能直接运行
   ↓ 链接    gcc 或 ld
可执行文件           二进制文件，可以运行

```
预处理器 cpp：处理 #include、#define、条件编译
编译器 cc1：把 C 代码翻译成汇编代码
汇编器 as：把汇编代码翻译成目标文件 .o
链接器 ld：把 .o 文件和库文件连接成可执行文件
```

因此

```
.c  →    .i  →        .s  →      .o    →  可执行文件
源程序  预处理文件  汇编文件  目标文件    可执行文件
文本    文本        文本      二进制      二进制
```

# GCC参数

1. -E:预处理

`gcc -E -o eg.i eg.c`

只进行预处理，生成 .i 文件。
不会编译、不会汇编、不会链接。

处理

#include
#define
#ifdef / #if
宏替换
删除注释

2. -S：生成汇编文件

`gcc -S -o eg.s eg.c`

把 C 源程序编译成汇编语言文件 .s。
到汇编阶段前停止，不生成 .o，也不生成可执行文件。

如果要intel风格：`gcc -S -masm=intel -o eg.s eg.c`

3. -C:只编译/汇编，不连接

`gcc -c -o eg.o eg.c`

生成目标文件 .o。
不会进行链接。
不会生成可执行文件。

4. 无参数直接生成可执行

`gcc -o eg eg.c`

预处理 → 编译 → 汇编 → 链接

5. -O：执行输出文件名

`gcc -o eg eg.c`：输出文件名叫做eg

6. -og：优化

# 汇编和链接

汇编：把.s汇编语言翻译成.o目标

链接：把.o目标文件和库文件合并，生成可执行文件


# GAS汇编语法：GNU的汇编器，处理.s文件

框架

``` asm
.intel_syntax noprefix

.data
msg:
    .asciz "hello"

.text
.globl main
main:
    mov eax, 0
    ret
```

1. .text：代码段。表示后面的内容属于代码段，放指令。必须有
2. .data：数据段：表示后面的内容属于数据段，主要放已经初始化的数据
3. .bss:未初始化数据段：放未初始化全局变量
4. .globl:声明全局符号。例如`.globl main`把main声明未全局符号，让连接器能找到

```
程序要有代码段 .text
要有入口标号 main 或 _start
入口标号通常要用 .globl 声明
数据段 .data 不是必须的
```

## 

1. 标号：后面有冒号

```
main:
loop:
msg:
```

表示：代码位置、数据位置、跳转目标、变量地址

2. 不以分号结束，以行结束
3. 注释：#或者/*多行*/，不是;
4. 一般AT&T语法
   1. 特点
      1. 寄存器前面有%
      2. 立即数前面有$
      3. 源操作数在前，目的操作数在后
5. INtel
   1. 寄存器前无%
   2. 立即数直接写
   3. 操作数在前，源在后
6. 切换Intel用：`.intel_syntax noprefix`
7. GCC生成Intel格式汇编
   1. `gcc -S -masm=intel -o eg.s eg.c`

# 机器语言、汇编、高级的关系

1. cpu只能直接执行机器（0，1组成）
2. 汇编是机器语言的助记符
3. 高级语言需要编译器（比如c）

# 文本文件和二进制文件

文本：人类打开可以看懂。有.c,.i,.s。汇编高级预处理都是

二进制：不能直接当作普通文本看。.o,可执行，.a ,.so


# 源程序框架

```asm
.intel_syntax noprefix
.text
.globl main
main:
    mov eax, 0
    ret
```

代码段通常必须有
入口标号必须有
入口标号通常要用 .globl 声明
数据段不是必须的

# 说明性和执行性

1. 说明性也叫伪指令。例如.text之类，给汇编器看的
2. 执行性语句。如mov eax, 1，第二类是执行性语句，也就是汇编指令。它们不是伪指令，而是会被汇编器翻译成机器指令，最终由 CPU 执行。
