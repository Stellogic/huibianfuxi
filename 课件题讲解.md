# 课件题讲解（题干复述版）

依据：课件 PDF 的可抽取文本层、总复习课件、课堂练习页和实验题页。  
口径：编程题默认按当前课程主线 `x86-64 + GAS + Intel syntax`。  
格式：每题都先写“题干复述”，再写“讲解/答案/易错点”。

说明：第 1 章有一页只抽取到输出数据和问号，题目代码本身不在文本层中，无法可靠复述题干，因此没有强行编造。其余能从课件文本层识别出的题、练习、例题型题已经按考试价值整理。

## 0. 先看这些最高频题型

1. 指令是否合法：内存到内存、操作数位宽、内存类型、立即数作目的操作数。
2. `list` / 数据存储：`.byte/.word/.long/.quad`、小端、`.align`、偏移地址。
3. C 与汇编互推：看参数寄存器、返回值、数组寻址、条件跳转。
4. 条件跳转：无符号 `above/below`，有符号 `greater/less`。
5. 除法：`EDX:EAX / r/m32 -> EAX` 商、`EDX` 余数。
6. 128 位或多精度：低位 `add/sub`，高位 `adc/sbb`。
7. 循环：`loop` 不改标志，`ECX=0` 会先减成 `0xffffffff`。
8. 子程序：`call` 压返回地址，`ret` 弹返回地址；栈参数看 `[rbp+16]` 起。
9. 宏：汇编时展开，不是运行时 `call`。

## 1. 课堂练习与测试题

### 题 1：判断下列传送类指令是否正确

**题干复述：** 课件给出 13 条指令，要求判断每条指令是否正确：

```asm
1  mov dvar[rip+4], dvar[rip]
2  mov edi, si
3  mov eax, 0x50a
4  mov al, 0x50a
5  mov dword ptr [rbx], 255
6  mov [rbx+4], 200
7  mov ds, 100
8  mov ax, ds
9  mov edx, [ebx+0xff]
10 mov [rsi], dword ptr [rdi]
11 mov ecx, cs
12 xchg al, byte ptr dvar[rip+3]
13 xchg 100, al
```

**答案：**

| 序号 | 对错 | 讲解 |
|---:|---|---|
| 1 | 错 | 两个操作数都是内存，典型 x86 指令不能内存到内存直接传送。 |
| 2 | 错 | `edi` 是 32 位，`si` 是 16 位，位宽不匹配。 |
| 3 | 对 | 立即数 `0x50a` 可以送入 32 位 `eax`。 |
| 4 | 错 | `al` 只有 8 位，`0x50a` 超出 8 位范围。 |
| 5 | 对 | 已写 `dword ptr`，类型明确，立即数写内存可以。 |
| 6 | 错 | 内存操作数类型不明确，应写 `byte/word/dword/qword ptr`。 |
| 7 | 错 | 不能直接把立即数送入段寄存器 `ds`。 |
| 8 | 对 | 可把段寄存器内容送入 16 位通用寄存器 `ax`。 |
| 9 | 对 | 课件按可接受寻址判断为正确。 |
| 10 | 错 | 源和目的都是内存，不能直接内存到内存。 |
| 11 | 错 | `cs` 是段寄存器，不能直接送入 32 位 `ecx`；若题目允许，应看 `cx`。 |
| 12 | 对 | `xchg` 可在寄存器和内存之间交换，类型也明确。 |
| 13 | 错 | 立即数不能作为 `xchg` 的操作数。 |

**易错点：** 指令合法性题优先检查 4 件事：是不是内存到内存、位宽是否匹配、内存类型是否明确、立即数是否当了目的操作数。

### 题 2：根据指令序列写出 `eax` 的公式

**题干复述：** 课件给出如下指令，问执行后 `eax` 中的值是什么，用公式表示。

```asm
movsx eax, cx
imul eax, 9
cdq
mov ebx, 5
idiv ebx
add eax, 32
```

**讲解：**

1. `movsx eax, cx`：把 16 位 `cx` 按有符号数扩展到 32 位，设为 `C`。
2. `imul eax, 9`：`eax = 9 * C`。
3. `cdq`：把 `eax` 符号扩展成 `edx:eax`，为有符号除法做准备。
4. `idiv ebx`：`(edx:eax) / 5`，商在 `eax`。
5. `add eax, 32`：商再加 32。

**答案：**

```text
EAX = (9 * (short)CX) / 5 + 32
```

有符号除法 `idiv` 的商向 0 截断。这题本质是摄氏温度转华氏温度：

```text
F = 9 * C / 5 + 32
```

### 题 3：移位与循环移位选择题

**题干复述：** 课件问：

1. 指令 `shl eax, 1` 执行后，`EAX` 的最低位一定是哪个选项？  
   A. 0  
   B. 1  
   C. 与最高位相同  
   D. 与进位 `CF` 标志相同

2. 指令 `ror eax, 1` 执行后，`EAX` 的最高位一定是哪个选项？  
   A. 0  
   B. 1  
   C. 与最低位相同  
   D. 与进位 `CF` 标志相同

**答案：**

1. `shl eax, 1`：选 A。左移 1 位，最低位补 0。
2. `ror eax, 1`：选 D。循环右移 1 位时，原最低位移到最高位，同时也进入 `CF`，所以新最高位与 `CF` 相同。

**易错点：** `ror` 是循环右移，不是逻辑右移；它不是高位补 0，而是把移出的低位绕回最高位。

### 题 4：指令位置寄存器与顺序执行判断

**题干复述：** 课件给出：

1. 多选题：x86 处理器用来指示程序中指令位置的寄存器是：  
   A. `DS`  
   B. `CS`  
   C. `RAX`  
   D. `RIP`

2. 判断题：程序顺序执行时，每执行一条指令，指令指针 `RIP` 就加 1。

**答案：**

1. 选 B、D。传统上指令位置由代码段寄存器 `CS` 和指令指针 `IP/EIP/RIP` 配合表示；在 x86-64 主线里重点看 `RIP`。
2. 判断题是错的。顺序执行时，`RIP` 变成下一条指令的地址，但不是固定加 1，而是加上当前指令的机器码长度；遇到跳转、调用、返回时还会改成目标地址。

### 题 5：`jmp nvar` 的指令寻址方式

**题干复述：** 已知 `nvar` 是一个变量，指令 `jmp nvar` 采用的指令寻址方式是：  
A. 相对寻址  
B. 直接寻址  
C. 存储器间接寻址  
D. 寄存器间接寻址

**答案：** C，存储器间接寻址。

**讲解：** `nvar` 是变量名，代表内存位置。`jmp nvar` 的目标地址来自内存单元中保存的内容，而不是直接把 `nvar` 这个标号当目标地址跳过去。若写 `jmp label`，那才更像直接按标号转移。

### 题 6：条件转移指令选择题

**题干复述：**

   A. `SF=1`  
   B. `PF=1`  
   C. `OF=1`

**答案：**

1. `JC`：选 D，`CF=1`。

### 题 7：`loop` 指令判断题

**题干复述：** 课件给出 4 个判断题：

1. 当 `ecx=0` 时，`loop` 指令执行循环体的次数为 0。
2. 控制循环是否结束只能安排在循环体之后。
3. `LOOP` 指令相当于 `DEC ECX` 和 `JNZ` 两条指令的功能，但考虑对状态标志的影响，它们是有差别的。
4. 当 `ecx=1` 时，执行 `LOOP` 指令不会发生转移。

**答案：**

1. 错。`loop` 会先把 `ecx` 减 1，`ecx=0` 会变成 `0xffffffff`，然后非 0，发生转移。
2. 错。循环控制可以放在循环前、循环后，也可以用多处条件控制。
3. 对。功能近似是 `ecx--` 后非 0 跳转，但 `loop` 不影响标志位，`dec` 会影响部分标志位。
4. 对。`ecx=1` 执行 `loop` 后减为 0，不发生转移。

**易错点：** 进入 `loop` 循环前若可能 `ecx=0`，通常先写：

```asm
jecxz done
```

### 题 8：`.global varname` 的含义

**题干复述：** 某程序模块使用伪指令 `.global varname`，这表明 `varname`：  
A. 是共享的，其他程序模块也可以使用  
B. 是在其他程序模块中定义的  
C. 是在本程序模块定义的  
D. 只能在本程序中使用，不能定义  
E. 既可以在本程序定义，也可以进行使用

**答案：** A、C、E。

**讲解：** `.global/.globl varname` 是把符号导出给链接器。通常这个符号在本模块定义，同时允许其他模块引用。它不表示“这个符号一定在其他模块定义”，那更接近 `.extern` 的语义。

### 题 9：总复习程序阅读 `counta`

**题干复述：** 课件给出程序：主程序把字符串地址和长度压栈，调用子程序 `counta`，子程序统计字符串中大写字母 `A` 的数量。要求填空 `(1)~(5)`，并画出进入子程序、压入寄存器保护后的堆栈区域。

```asm
; 数据段
msg: .asciz "Try to make it."
.equ count, (. - msg)

; 主程序
push count
lea rax, msg[rip]
push rax
call counta
add rsp, (1)
call dispuid

counta:
push rbp
mov rbp, rsp
push rbx
push rcx
mov rbx, (2)
mov rcx, (3)
mov rax, 0
again:
    cmp [rbx], 'A'
    jne next
    inc rax
next:
    inc rbx
    loop again
pop rcx
(4)
(5)
ret
```

**答案：**

```asm
(1) 16
(2) [rbp+16]
(3) [rbp+24]
(4) pop rbx
(5) pop rbp
```

更严谨地写比较字节时可写：

```asm
cmp byte ptr [rbx], 'A'
```

**讲解：**

主程序压栈顺序：

```asm
push count        # 先压长度
push msg_address  # 后压字符串地址
call counta       # 再压返回地址
```

进入子程序后：

```asm
push rbp
mov rbp, rsp
```

所以：

```text
[rbp]    = 旧 rbp
[rbp+8]  = 返回地址
[rbp+16] = 字符串地址 msg
[rbp+24] = count
```

子程序又压入：

```asm
push rbx
push rcx
```

此时以 8 字节为单位，堆栈含义可画成：

```text
低地址
[rbp-16] 保存的 rcx
[rbp-8]  保存的 rbx
[rbp]    保存的旧 rbp    <- rbp 指向这里
[rbp+8]  返回地址
[rbp+16] msg 的地址
[rbp+24] count
高地址
```

### 题 10：`eg0214.c` 中变量如何解释

**题干复述：** 课件给出 C 变量：

```c
int x = -1;
unsigned int u = 2147483648;
long l = -1;
unsigned long ul = -1;
short s = -1;
unsigned short us = -1;
char c = 'A';
unsigned char uc = 200;
char str[] = "Hello Assembly!";
char str1[] = {'C','h','i','n','a','\0'};
char *ps = str;
```

并用 `%x/%d/%u/%ld/%llu/%c/%s` 等格式显示，要求理解同一二进制数据按不同类型输出的结果。

**讲解：**

同一串二进制位，按有符号、无符号、字符、字符串解释会得到不同结果。

重点例子：

```text
int x = -1
32 位补码 = 0xffffffff
按 %d 输出：-1
按 %u 输出：4294967295
按 %x 输出：ffffffff
```

```text
unsigned int u = 2147483648
二进制 = 0x80000000
按无符号解释：2147483648
按有符号 int 解释：-2147483648
```

```text
char c = 'A'
ASCII = 0x41 = 65
按 %c 输出：A
```

```text
unsigned char uc = 200
字节 = 0xc8
按无符号解释：200
若按有符号 char 解释，可能是 -56
```

**易错点：** 不是数据变了，而是解释方式变了。

### 题 11：变量存储 `bvar/wvar/dvar`

**题干复述：** 课件给出：

```asm
bvar: .byte 0x39, 0x31, 0x32, 0x38
wvar: .word 0x3139, 0x3832
dvar: .long 0x38323139
```

要求分析这些数据在内存中的存储顺序。

**答案：**

`bvar` 是逐字节定义，不涉及多字节拆分：

```text
低地址 -> 高地址：39 31 32 38
```

`wvar` 中每个 `.word` 2 字节，小端存储：

```text
0x3139 -> 39 31
0x3832 -> 32 38
低地址 -> 高地址：39 31 32 38
```

`dvar` 中 `.long` 4 字节，小端存储：

```text
0x38323139 -> 39 31 32 38
低地址 -> 高地址：39 31 32 38
```

**结论：** 这三个定义在内存里的 4 个字节可以完全相同，但汇编源代码的“数据类型”和解释方式不同。

### 题 12：list 文件中的小端显示

**题干复述：** 课件 `eg0214.lst` 中出现：

```asm
ps:
0000 10000000 .quad str
     00000000

str:
0010 48656C6C .ascii "Hello Assembly!\0"
     6F204173
     73656D62
     6C792100

0028 FFFFFFFF .long -1
0030 00000080 .long -2147483648
```

要求理解偏移地址、目标代码、字符串和小端存储。

**讲解：**

1. `.quad str` 占 8 字节，保存的是 `str` 的地址。若 `str` 偏移是 `0x10`，小端字节就是：

```text
10 00 00 00 00 00 00 00
```

2. `.ascii "Hello Assembly!\0"` 按字符逐字节存储：

```text
H e l l o   A s s e m b l y ! \0
48 65 6C 6C 6F 20 41 73 73 65 6D 62 6C 79 21 00
```

3. `.long -1` 的 32 位补码是：

```text
0xffffffff
```

4. `.long -2147483648` 的 32 位补码是：

```text
0x80000000
```

按小端实际字节为：

```text
00 00 00 80
```

所以 list 里可能显示成 `00000080` 这种目标字节顺序。

## 3. 传送、寻址、算术运算题

### 题 13：`lea` 实现算术表达式

**题干复述：** 课件给出 C 函数：

```c
int scale(int x, int y, int z)
{
    long t = x + 4*y + 12*z;
    return t;
}
```

并给出参数对应：`x` 在 `rcx`，`y` 在 `rdx`，`z` 在 `r8`，要求理解 `lea` 指令如何实现算术。

**讲解与答案：**

课件汇编：

```asm
lea eax, [rcx+rdx*4]    # eax = x + 4*y
lea edx, [r8+r8*2]      # edx = 3*z
lea eax, [rax+rdx*4]    # eax = x + 4*y + 12*z
```

关键：

```text
r8 + r8*2 = 3z
(3z) * 4 = 12z
```

所以最终：

```text
EAX = x + 4y + 12z
```

**易错点：** `lea` 虽然名字是“取有效地址”，但常被编译器用来做不影响标志位的整数加法和乘小常数。

### 题 14：堆栈操作示例

**题干复述：** 课件给出：

```asm
.equ ten, 10
dvar: .quad 0x67762000, 0x12345678

mov eax, dvar[rip+4]       # EAX = ?
push rax
push qword ptr ten
push dvar[rip]
pop rax
pop dvar[rip+4]
mov rbx, dvar[rip+4]       # RBX = ?
pop rcx
```

要求分析栈顶变化和寄存器/内存结果。

**讲解：**

初始 `dvar` 是两个 8 字节数。小端下：

```text
dvar[rip]   = 0x0000000067762000
dvar[rip+8] = 0x0000000012345678
```

注意：按课件文本里的 `.quad` 严格推算，`dvar+4` 是第一个 8 字节数的高 4 字节；如果读 `dvar+8` 的低 4 字节，才会得到 `0x12345678`。课件这一页的注释更像是在强调“按偏移取数”和“栈后进先出”，考场上要以题面真实数据定义和偏移为准，不能机械背注释。

如果按课件注释口径，`mov eax, dvar[rip+4]` 给出：

```text
EAX = 0x12345678
```

三次压栈后，栈顶依次弹出：

1. `pop rax`：弹出最后压入的 `dvar[rip]`。
2. `pop dvar[rip+4]`：把 `ten=10` 写到 `dvar+4` 位置。
3. `mov rbx, dvar[rip+4]`：读出刚写入的值，所以：

```text
RBX = 0x000000000000000A
```

最后 `pop rcx` 弹出最早压入的 `rax`。

**易错点：** 栈是后进先出；64 位 `push/pop` 默认按 8 字节影响 `rsp`。

### 题 15：`pushf/pop rax` 查看标志寄存器

**题干复述：** 课件例题用：

```asm
pushf
pop rax
call disphx
```

显示 `RFLAGS` 的值，图中示例显示为 `0000000000000202`。要求理解 `pushf/pop` 对栈和 `rax` 的影响。

**讲解：**

`pushf/pushfq` 把标志寄存器压入栈：

```text
rsp = rsp - 8
[rsp] = RFLAGS
```

`pop rax` 再把栈顶弹入 `rax`：

```text
rax = [rsp]
rsp = rsp + 8
```

所以 `rax` 中得到当前 `RFLAGS` 的值。课件示例 `0x202` 里通常包含固定为 1 的位和中断允许等状态位。

### 题 16：无符号 64 位二进制加法

**题干复述：** 课件给出两个 64 位数，用两个 `.long` 表示低 32 位和高 32 位：

```asm
dvar1: .long 0x82347856, 0x67783000
dvar2: .long 0x12348998, 0x67762000
dvar3: .quad 0
```

要求用 32 位加法实现 64 位无符号加法。

**讲解与答案：**

低 32 位先加：

```asm
mov eax, [r8]
add eax, [r9]
mov [r10], eax
```

高 32 位再加，并接上低位产生的进位 `CF`：

```asm
mov eax, [r8+4]
adc eax, [r9+4]
mov [r10+4], eax
```

课件给出的中间结果：

```text
低位：0x82347856 + 0x12348998 = 0x946901ee，CF=0
高位：0x67783000 + 0x67762000 + CF = 0xceee5000
```

最终 64 位结果：

```text
0xceee5000946901ee
```

### 题 17：64 位二进制减法

**题干复述：** 课件给出：

```asm
dvar1: .long 0x82347856, 0x67783000
dvar2: .long 0x12348998, 0x67762000
dvar3: .quad 0
```

要求用 32 位减法实现 64 位减法。

**讲解与答案：**

低 32 位先减：

```asm
mov eax, [r8]
sub eax, [r9]
mov [r10], eax
```

高 32 位再减，并接上低位借位 `CF`：

```asm
mov eax, [r8+4]
sbb eax, [r9+4]
mov [r10+4], eax
```

课件给出：

```text
低位结果：0x6fffeebe
高位结果：0x00021000
```

最终：

```text
0x000210006fffeebe
```

**口诀：** 加法 `add + adc`，减法 `sub + sbb`。

### 题 18：无符号除法中为什么保存除数

**题干复述：** 课件给出 C 函数：

```c
void eg0414a(unsigned long x, unsigned long y,
             unsigned long *qp, unsigned long *rp)
{
    unsigned long q = x / y;
    unsigned long r = x % y;
    *qp = q;
    *rp = r;
}
```

参数约定：`ecx=x`，`edx=y`，`r8=qp`，`r9=rp`。汇编片段：

```asm
mov eax, ecx
mov r10d, edx     # y保存到r10d，为什么？
mov edx, 0
div r10d
mov DWORD PTR [r8], eax
mov DWORD PTR [r9], edx
```

**答案：**

因为 32 位 `div` 的被除数固定使用 `EDX:EAX`。执行除法前必须把 `edx` 清 0 作为被除数高 32 位：

```asm
mov edx, 0
```

但 `edx` 原本保存除数 `y`，如果不先复制到 `r10d`，除数就丢了。因此要：

```asm
mov r10d, edx
```

除法后：

```text
EAX = 商 q
EDX = 余数 r
```

### 题 19：有符号除法 `cdq/idiv`

**题干复述：** 课件给出 C 函数：

```c
void eg0414b(long x, long y, long *qp, long *rp)
{
    long q = x / y;
    long r = x % y;
    *qp = q;
    *rp = r;
}
```

参数约定：`ecx=x`，`edx=y`，`r8=qp`，`r9=rp`。汇编：

```asm

mov r10d, edx
cdq
idiv r10d
mov DWORD PTR [r8], eax
mov DWORD PTR [r9], edx
```

**讲解：**


```text

```

也就是把 32 位有符号被除数准备成 64 位的 `EDX:EAX`。

### 题 25：截取整数中的 m 位

**题干复述：** 课件例题：设整数占两个字节，求一个整数 `num` 从右端第 `n` 位开始的 `m` 位，其中 `0 <= n <= 15`，`m > 0` 且 `n + m <= 16`。

**答案：**

C 表达式：

```c
(num >> n) & (~(~0 << m))
```

**讲解：**

1. `num >> n`：把目标字段移到最低位。
2. `~0 << m`：低 `m` 位为 0，其余为 1。
3. `~(~0 << m)`：生成低 `m` 位全 1 的掩码。
4. 最后按位与，保留低 `m` 位。

例：要取 4 位，掩码就是：

```text
0000 0000 0000 1111
```

### 题 29：判断 `eax` 的 D1 位和设置校验位

**题干复述：** 课件实例：

1. 判断 `eax` 的 D1 位是否为 0：

```asm
ror eax, 2
jnc L2
...
L2:
rol eax, 2
```

2. 设置校验位：

```asm
and al, 0x7f
jnp next
or al, 0x80
next:
```

**讲解：**

第 1 问：`ror eax, 2` 后，原来的 D1 位会进入 `CF`。  
所以：

```asm
jnc L2
```

表示 `CF=0` 时跳转，也就是原 D1 位为 0。最后 `rol eax, 2` 恢复原来的位序。

第 2 问：

```asm
and al, 0x7f
```

先把最高位清 0，并根据低 7 位结果更新 `PF`。`PF=1` 表示低 8 位中 1 的个数为偶数，`jnp` 表示 `PF=0`，也就是个数为奇数。

若已有奇数个 1，就不用设置最高位；否则：

```asm
or al, 0x80
```

把最高位置 1，让 1 的个数变成奇数。

### 题 35：地址表实现四则运算计算器

**题干复述：** 课件例题要求编制一个简单四则运算计算器：输入规定符号，实现两个有符号数加、减、乘、除、移位等运算；输入其他字符则提示非法并退出。C 逻辑为：

```c

    switch(n) {
    case '+': *result = x + y; break;
    case '-': *result = x - y; break;
    case '*': *result = x * y; break;

```

**讲解：**

```asm
cmp cl, 20       # '>' - '*' = 20
```

```asm
.L4:

    ...
```

此时取表：

```asm
mov rax, QWORD PTR [r10+rcx*8]
jmp rax
```

**易错点：** 地址表题的核心不是背每个标号，而是会解释：

```asm
add r9d, DWORD PTR -4[rcx+r8*4]   # + fibs[i-1]
mov DWORD PTR [rcx+r8*4], r9d     # fibs[i]
```

因为 `int` 是 4 字节，所以偏移是 `i*4`。

### 题 42：`LOOP` 版阶乘的问题和修正

**题干复述：** 课件给出 `LOOP` 版阶乘：

```asm
mov eax, 1
mov ecx, N[rip]
again:
    imul eax, ecx
    loop again
```

并指出 `N=100` 时结果错误，后续又指出 `N=0` 会出现死循环。要求分析原因并修正。

**讲解：**

`N=100` 错误：阶乘很快超出 32 位，`eax` 溢出。课件尝试用：

```asm
cmp eax, 0
js L1
```

检测结果变负，但这只是粗略检测。更通用的有符号溢出应看 `OF`，例如 `jo`。

`N=0` 死循环：

```asm
loop again
```

当 `ecx=0` 时会先减为 `0xffffffff`，然后跳转，造成超大循环。

修正入口：

```asm
cmp ecx, 0
jz L1
```

或：

```asm
jecxz L1
```

### 题 43：求数组最大值

### 题 44：异或加密解密

```asm
    xor byte ptr [rsi+rbx], al
```

这样直接用 `rcx` 当下标，不需要额外 `rbx`。

### 题 49：十六进制显示 32 位数

**题干复述：** 课件要求利用字符串显示子程序 `DISPMSG` 实现 32 位数的十六进制显示，例如显示 `EAX=1234ABCD` 这类形式。

**讲解：**

32 位数有 8 个十六进制位，每 4 位转换成 1 个字符。

单个十六进制数字转 ASCII：

```text
0~9：加 0x30 -> '0'~'9'
A~F：先加 0x30 后会超过 '9'，再加 7 -> 'A'~'F'
```

算法：

1. 取最高或最低 4 位。
2. 转成 ASCII。
3. 保存到预留字符串位置。
4. 重复 8 次。
5. 调用显示字符串子程序。

### 题 50：有符号十进制显示

**题干复述：** 课件要求以十进制形式显示 32 位二进制数的真值，并给出转换算法。

**讲解：**

算法步骤：

1. 判断数据是 0、正数还是负数。
2. 若为 0，直接显示 `"0"`。
3. 若为负数，先显示 `'-'`，再求绝对值。
4. 反复除以 10，余数加 `0x30` 转成 ASCII 数字。
5. 因为先得到低位数字，所以保存后要从高位到低位显示。

例：

```text
123 / 10 = 12 ... 3 -> '3'
12  / 10 = 1  ... 2 -> '2'
1   / 10 = 0  ... 1 -> '1'
```

显示顺序是：

```text
1 2 3
```

### 题 51：二进制数字符串转二进制数

**题干复述：** 课件要求把二进制数字符串转换为二进制数，写成子程序。入口参数 `rdx` 是二进制数字符串首地址，出口参数用共享变量 `temp` 返回。

**讲解：**

核心算法：

1. 读一个字符，必须是 `'0'` 或 `'1'`。
2. 字符减 `0x30`，得到数值 0 或 1。
3. 原结果左移 1 位。
4. 把新位合并进去。
5. 重复直到字符串结束。

伪代码：

```c
value = 0;
while (*p) {
    bit = *p - '0';
    value = (value << 1) | bit;
    p++;
}
```

### 题 53：堆栈传参求平均值

**题干复述：** 课件例题要求计算 32 位有符号数组平均值。主程序先压入数据个数 `len`，再压入数组首地址，调用 `mean`，子程序返回 `EAX=平均值`。

主程序：

```asm
push len
lea rax, array[rip]
push rax
call mean
add rsp, 16
```

子程序：

```asm
mean:
    push rbp
    mov rbp, rsp
    push rbx
    push rcx
    push rdx
    mov rbx, [rbp+16]
    mov rcx, [rbp+24]
    xor eax, eax
    xor rdx, rdx
```

**讲解：**

压栈后参数位置：

```text
[rbp+16] = 数组首地址
[rbp+24] = 元素个数 len
```

循环求和：

```asm
mean1:
    add eax, [rbx+rdx*4]
    add edx, 1
    cmp edx, ecx
    jb mean1
```

求平均值：

```asm
cdq
idiv ecx
```

返回前恢复寄存器：

```asm
pop rdx
pop rcx
pop rbx
pop rbp
ret
```

主程序 `add rsp, 16` 是平衡两个 8 字节参数，也可由子程序使用 `ret 16` 实现类似效果。

### 题 54：`swap_add` 与 `caller`

**题干复述：** 课件给出 C 函数：

```c
long swap_add(long *xp, long *yp)
{
    long x = *xp;
    long y = *yp;
    *xp = y;
    *yp = x;
    return x + y;
}
```

以及调用者：

```c
long caller()
{
    long arg1 = 534;
    long arg2 = 1057;
    long sum = swap_add(&arg1, &arg2);
    long diff = arg1 - arg2;
    return sum * diff;
}
```

要求理解指针参数、局部变量和栈布局。

**讲解：**

`swap_add` 中：

```asm
mov eax, DWORD PTR [rcx]     # x = *xp
mov r8d, DWORD PTR [rdx]     # y = *yp
mov DWORD PTR [rcx], r8d     # *xp = y
mov DWORD PTR [rdx], eax     # *yp = x
add eax, r8d                 # return x+y
ret
```

在 Win64 口径下：

```text
rcx = xp
rdx = yp
eax = 返回值
```

`caller` 中：

```asm
sub rsp, 56
mov DWORD PTR 44[rsp], 534
mov DWORD PTR 40[rsp], 1057
lea rdx, 40[rsp]     # &arg2
lea rcx, 44[rsp]     # &arg1
call swap_add
```

`arg1/arg2` 放在栈上，是因为要把它们的地址传给 `swap_add`。

### 题 56：状态标志显示宏 `rfbit`

**题干复述：** 课件例题要求用宏显示 `OF, SF, ZF, AF, PF, CF`。宏定义：

```asm
.macro rfbit bit1, bit2
    xor ebx, ebx
    rol eax, \bit1
    adc ebx, 0x30
    mov rfmsg[rip+\bit2], bl
.endm
```

宏调用：

```asm
rfbit 21, 3
rfbit 4, 9
rfbit 1, 15
rfbit 2, 21
rfbit 2, 27
rfbit 2, 33
```

要求理解宏展开和标志位转字符。

**讲解：**

宏的目的：

1. 通过 `rol eax, bit1` 把目标标志位移入 `CF`。
2. `adc ebx, 0x30` 计算字符：

```text
CF=0 -> ebx = '0'
CF=1 -> ebx = '1'
```

3. 把字符写入 `rfmsg` 指定位置。

宏调用不是运行时 `call`，而是汇编时展开。例如：

```asm
rfbit 21, 3
```

会展开成：

```asm
xor ebx, ebx
rol eax, 21
adc ebx, 0x30
mov rfmsg[rip+3], bl
```

### 题 57：宏与子程序如何选用

**题干复述：** 课件问宏与子程序的选用原则。

**答案：**

选宏：

- 程序段较短。
- 要求执行较快。
- 适合全局性预处理或简单重复代码。
- 不需要运行时 `call/ret`。

选子程序：

- 程序段较长。
- 希望减小目标代码体积。
- 希望模块更独立，便于分别编写、链接和复用。

**对比：**

| 宏 | 子程序 |
|---|---|
| 汇编时展开 | 运行时调用 |
| 不用 `call/ret` | 用 `call/ret` |
| 每调用一次展开一份 | 通常只有一份代码 |
| 速度快但可能增大代码 | 节省代码但有调用开销 |

### 题 65：实验七显示内存区域 `DISPMEM`

**题干复述：** 利用十六进制字节显示子程序 `DISPHB`，设计从低地址到高地址逐个字节显示某个主存区域内容的子程序 `DISPMEM`。入口参数：`EAX=主存偏移地址`，`ECX=字节个数`。主程序验证字符串 `"This is a test!"`。

**讲解：**

核心循环：

```asm
; rbx = 起始地址
; ecx = 字节数
again:
    mov al, byte ptr [rbx]
    call DISPHB
    inc rbx
    loop again
```

注意入口参数按课件写的是 `EAX/ECX`，当前 x86-64 编程时地址更稳妥用 `RAX/RBX` 保存。

### 题 67：实验七 Eratosthenes 筛法

**题干复述：** 使用 Eratosthenes 筛法，求 `1~100000` 之间共有多少个素数。算法要求创建字节数组：从 2 开始，把所有 2 的倍数标记；再处理 3 的倍数；再处理 5 的倍数；如此重复，最后没有被标记的位置对应素数。

**讲解：**

伪代码：

```c
for (i = 0; i <= 100000; i++)
    mark[i] = 0;

for (p = 2; p*p <= 100000; p++) {
    if (mark[p] == 0) {
        for (k = p*p; k <= 100000; k += p)
            mark[k] = 1;
    }
}

count = 0;
for (i = 2; i <= 100000; i++)
    if (mark[i] == 0) count++;
```

汇编重点：

- 字节数组用 `.space 100001`。
- `mark[k]` 地址是 `base + k`。
- 内层循环步长是 `p`。

### 题 68：实验八模块化平均值程序

**题干复述：** 编写数据输入、求平均值以及输出程序。使用有符号十进制数据输入、求平均值、输出子程序，实现从键盘输入 10 个数据，求平均值，并输出平均值。要求主程序 `exp0801.s` 定义：

```asm
msg1: .string "Enter 10 numbers:\n"
msg2: .string "The mean is:"
```

调用子程序输入 10 个数据、求平均值、输出；子程序写在 `exp0801s.s`；说明模块连接过程；再形成子程序库 `libmysub.a`。输入数据为：

```text
1234567890, -1234, 0, 1, -987654321, 32767, -32768, 5678, -5678, 9000
```

**讲解：**

主程序结构：

```asm
.intel_syntax noprefix
.data
msg1: .string "Enter 10 numbers:\n"
msg2: .string "The mean is:"
array: .space 40

.text
.globl main
main:
    # 显示 msg1
    # 调 input 子程序读 10 个数到 array
    # 调 mean 子程序求平均值
    # 显示 msg2 和平均值
    mov eax, 0
    ret
```

模块连接流程：

```text
exp0801.s  -> exp0801.o
exp0801s.s -> exp0801s.o
链接 exp0801.o + exp0801s.o + I/O 库 -> 可执行文件
```

子程序库：

```text
多个 .o 打包成 .a
链接时把 .a 加入命令
```

### 题 69：实验八逻辑指令宏 `LOGICAL`

**题干复述：** 定义一个使用逻辑指令的宏 `LOGICAL`。主程序不能改动，需要补充两个宏定义：

```asm
LOGICAL AND, 0xFF, 0x0F
LOGICAL OR,  0xF0, 0x0F
LOGICAL XOR, 0xFF, 0x0F
LOGICAL TEST, 0xFF, 0x0F
LOGICAL_NOT 0
```

`LOGICAL` 代表 `AND/OR/XOR/TEST` 四条双操作数逻辑指令；`LOGICAL_NOT` 代表单操作数 `NOT`。注意参数区分。

**参考答案：**

```asm
.macro LOGICAL op, a, b
    mov eax, \a
    \op eax, \b
.endm

.macro LOGICAL_NOT a
    mov eax, \a
    not eax
.endm
```

**展开示例：**

```asm
LOGICAL AND, 0xFF, 0x0F
```

展开为：

```asm
mov eax, 0xFF
AND eax, 0x0F
```

实际 GAS 对助记符大小写通常不敏感，但考试手写建议统一小写：

```asm
and eax, 0x0F
```

**易错点：**

- `NOT` 只有一个操作数，不能套用双操作数宏。
- 宏不是子程序，不需要 `call` 和 `ret`。

## 9. 最后背法

1. 先背第 1 节：选择/判断/改错，最像客观题。
2. 再背第 2、3 节：list、小端、寻址、算术，最像分析题。

```text

```
