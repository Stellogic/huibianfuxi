# x86-64寻址方式

## 1. 内存寻址形式

`[base + index * scale + displacement]`

有效地址 = 基址寄存器 + 变址寄存器 × 比例因子 + 偏移量

## 2. lea:不访问内存，只计算地址表达式的值

`lea rdx, [rax + rax*8 + 7]`

## 4.BYTE PTR

`INC BYTE PTR [RSI]`

INC        加 1
BYTE PTR   说明这次操作的是 1 个字节
[RSI]      RSI 里面存的是地址，去这个地址访问内存

同理：WORD PTR就是2字节，DWORD就是4字节，QUORD就是8字节

# mov 目的操作数，源操作数

mov不允许普通存储器到存储器

`mov [rax], [rbx]`

这个一般不合法x86 普通指令通常不允许两个操作数都来自内存。

实现内存到内存需要中专寄存器

```asm
mov eax, [rbx]
mov [rax], eax
```

# PUSH和POP栈操作栈

栈是内存区域用来保存临时数据：比如局部变量、函数返回地址


栈向低地址方向增长

所以

push：往栈里放东西，RSP（栈指针寄存器） 变小
pop ：从栈里取东西，RSP 变大

**RSP指向栈顶**

比如

RSP = 0x1000
RAX = 0x1234
push rax

因为rax是8字节

RSP = RSP - 8

RSP = 0x0FF8

之后再内存[0x0FF8] = 0x1234

因此：`push rax`等价于`RSP = RSP - 8
[RSP] = RAX`

同理 `pop rdx`等价于`RDX = [RSP]
RSP = RSP + 8`
