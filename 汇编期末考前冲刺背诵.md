## 1. 开发流程与工具命令

### 1.1 四步流程

| 步骤       | 工具                                                                 | 产生文件                               |
| ---------- | -------------------------------------------------------------------- | -------------------------------------- |
| 编辑       | 文本编辑器 / IDE（Integrated Development Environment，集成开发环境） | 源程序 `.s` / `.c`                    |
| 汇编或编译 | `as` / `gcc -S` / `gcc -c`                                           | 汇编文件 `.s` 或目标文件 `.o`         |
| 连接       | `ld` / `gcc`                                                         | 可执行文件                             |
| 调试运行   | `gdb` / 操作系统                                                     | 不产生主要新文件                       |

### 1.2 C 到可执行文件

```text
.c  --gcc -E-->  .i  --gcc -S-->  .s  --gcc -c/as-->  .o  --gcc/ld-->  可执行文件
文本              文本             文本              二进制            二进制
```

CPU 能直接识别和执行的是**机器语言/机器码**，汇编语言源程序必须经汇编器翻译成目标程序后才能执行。

课件原词：`gcc -E` 做预处理，常由预处理器 `cpp` 处理 `#include/#define` 等，生成 `.i`；`gcc -S` 编译成 `.s`；`as/gcc -c` 汇编成 `.o`；`ld/gcc` 连接成可执行文件。

必背命令：

```bash
gcc -E -o eg.i eg.c
gcc -S -Og -g -masm=intel -o eg.s eg.c
gcc -c -o eg.o eg.s
gcc -o eg eg.o ./lib/winio64.a
objdump -d -M intel eg.o
```

生成列表文件：

```bash
gcc -Og -masm=intel -Wa,-al -g ./progs/eg0214.c > eg0214.lst
as -al=eg0214.lst -o eg0214.o eg0214.s
```

### 1.3 GDB（GNU Debugger）常用命令

调试题、实验题里常见，至少会看懂：

```gdb
gdb exp0201
b main          # 在 main 设置断点
r               # 运行
nexti           # 单步执行一条汇编指令，跳过函数调用
stepi           # 单步执行一条汇编指令，进入函数调用
c               # 继续运行
p x             # 按默认格式查看变量
p/x $rax        # 以十六进制查看寄存器
p &x            # 查看变量地址
x/4b &x         # 从变量地址开始查看 4 个字节
disas /r main   # 反汇编并显示机器码
set disassembly-flavor intel
```

具体例子：观察变量、小端存储和反汇编。

```c
// exp0201.c
int main(void)
{
    int x = 0x12345678;
    int y = x + 5;
    return y;
}
```

编译时带 `-g`：

```bash
gcc -Og -g -masm=intel -o exp0201 exp0201.c
```

进入 GDB 后可这样看：

```gdb
gdb exp0201
(gdb) set disassembly-flavor intel
(gdb) b main
(gdb) r
(gdb) n                 # 执行完 int x = 0x12345678;
(gdb) p x               # 十进制显示 x，结果为 305419896
(gdb) p/x x             # 十六进制显示 x，结果为 0x12345678
(gdb) p &x              # 查看 x 的地址，如 0x7fffffffdabc
(gdb) x/4xb &x          # 从 &x 开始看 4 个字节
0x7fffffffdabc: 0x78 0x56 0x34 0x12
(gdb) disas /r main     # 看 main 的汇编和机器码
(gdb) nexti             # 单步执行一条机器指令
(gdb) p/x $eax          # 看 EAX，此处常用于观察 return y 的返回值
```

解释：`x = 0x12345678`，但内存从低地址到高地址显示为 `78 56 34 12`，这就是 x86-64 小端存储。`p/x` 适合看变量或寄存器的十六进制值；`x/4xb &x` 适合直接看内存字节。

记忆点：编译时加 `-g` 才方便源码级调试；看小端存储用 `x/xb`、`x/4b`、`x/4xb` 这类内存查看命令。

### 1.4 GAS（GNU Assembler）程序框架

```asm
.intel_syntax noprefix

.data
msg: .asciz "hello\n"

.text
.globl main
main:
    lea rax, msg[rip]
    call dispmsg
    mov eax, 0
    ret
```

背：

- `.text` 代码段，通常必须有。
- `.data` 已初始化数据段，不是必须。
- `.bss` 未初始化数据段。
- `.globl` / `.global` 声明全局符号。
- Intel 语法：目的在前，源在后；寄存器无 `%`，立即数无 `$`。
- GAS 注释：`#` 单行，`/* ... */` 多行；不是分号。

## 2. ISA（Instruction Set Architecture）与 x86-64

### 2.1 ISA 是什么

ISA（Instruction Set Architecture，指令集架构）是软件和硬件之间的**接口契约**，规定：

- 机器指令、指令格式、操作数类型。(cpu支持哪些命令以及怎么编码)
- 寄存器、标志位/条件码。（能用哪些寄存器以及标志位）
- 寻址方式、存储模型。（怎么找操作数，内存怎么组织）
- 中断/异常、I/O（Input/Output，输入/输出）方式等。

```
我能执行哪些指令？
我有哪些寄存器？
一条指令怎么写？
操作数可以来自哪里？
内存怎么访问？
出错或中断时怎么处理？
```

不属于 ISA：具体控制器实现、编译器、汇编器、连接器（这些是把高级语言或汇编转换成机器码的工具）。（关键看软件能不能“看见”和“依赖”？能看见依赖的）

如果是“程序员/编译器写代码时必须知道的 CPU 规则”，属于 ISA。

如果是“CPU 内部怎么造”或“软件工具怎么翻译”，不属于 ISA。

对汇编语言程序员来说，常把硬件抽象为寄存器、存储器地址、I/O 地址；系统总线这类内部连接结构不是主要编程抽象。

### 2.2 当前课程版本

本课程默认版本：

```text
x86-64 架构
GAS（GNU Assembler）汇编器
Intel 语法
GNU 工具链：gcc / as / ld / objdump / gdb
```

当前版本答题口径：

- x86-64 属于 x86 系列，指令长度可变，寻址方式多。
- x86-64 有 16 个 64 位通用寄存器：`RAX/RBX/RCX/RDX/RSI/RDI/RBP/RSP/R8-R15`。
- GAS Intel 语法中操作数顺序是“目的, 源”，寄存器不写 `%`，立即数不写 `$`。
- 64 位全局/静态数据访问常见 `label[rip]` 这种 RIP 相对寻址。
- 函数参数和栈对齐按 ABI（Application Binary Interface，应用程序二进制接口）判断：题面是 Win64 就按 Win64，题面是 Linux x86-64 就按 Linux x86-64。

### 2.3 旧题选择题补丁：8086、ARM/鲲鹏、Win/DOS

这些不是本届编程题主线，但单元测试和课本选择题里会出现，按选择题背即可。

#### 2.3.1 CISC 和 RISC

| 对比     | CISC/x86                         | RISC/ARM                         |
| -------- | -------------------------------- | -------------------------------- |
| 指令长度 | 通常可变                         | 通常固定，A64 常见 32 位定长     |
| 指令复杂度 | 指令多、格式多、寻址方式多       | 指令较精简、格式较少             |
| 访存特点 | 很多指令可以直接访问内存         | 典型 Load/Store 架构             |
| 代表     | x86/x86-64                       | ARM/AArch64/鲲鹏                 |

常见判断：x86 属于 CISC；ARM 属于 RISC；鲲鹏 920 基于 ARMv8。

#### 2.3.2 8086/实模式地址

实模式物理地址公式：

```text
物理地址 = 段地址 * 16 + 偏移地址 = 段地址 << 4 + 偏移地址
```

例：

```text
0x1460:0x0100 -> 0x14600 + 0x0100 = 0x14700
DS=0x1400, mov ax,[0x100] -> 0x14000 + 0x0100 = 0x14100
```

实地址方式常见限制：线性/物理地址空间最大 1MB，一个段最大 64KB。一个物理地址可以有多个逻辑地址表示，但物理地址本身唯一。

64 位程序中通常说程序员看到逻辑地址/线性地址/偏移地址，不直接使用物理地址。

#### 2.3.3 ARMv8/AArch64/鲲鹏

- ARM 是 RISC，AArch64 是 64 位执行状态，A64 是 AArch64 常用指令集。
- 按课件/单测口径：鲲鹏 920 基于 ARMv8；只支持 AArch64，不支持 AArch32。
- A64 触发系统调用用 `SVC #0`。
- AArch64 Linux 系统调用前 6 个参数常用 `x0-x5`；按本课程选择题，超过 6 个的额外参数通过栈传递。
- A64 十进制输出子程序遇到负数：先输出负号，再求补/取绝对值后输出数字。

ARM 异常等级和安全：

| 点       | 速记                                         |
| -------- | -------------------------------------------- |
| `EL0`    | 普通应用程序                                 |
| `EL1`    | 操作系统内核                                 |
| `EL2`    | 虚拟机监控程序/Hypervisor                    |
| `EL3`    | 安全监控/TrustZone 相关                      |
| `PSTATE` | 状态位逻辑统称，不是普通物理寄存器           |
| `SPSR_ELn` | 异常发生时保存当前 `PSTATE`                |
| `ELR_ELn` | 保存异常发生前的 `PC`，用于异常返回        |
| `FIQ`    | 快速中断，优先级高于普通 `IRQ`               |
| `vIRQ/vFIQ` | 虚拟化场景的虚拟中断，由虚拟机监控程序管理 |

安全世界/非安全世界选择题：安全世界可以访问安全世界和非安全世界资源；非安全世界不能访问安全世界资源。

#### 2.3.4 WinAPI、DOS、旧环境

- Win64 调 Windows API：要按调用约定设置参数，还要注意 32 字节影子空间和栈 16 字节对齐。
- `MessageBoxA` 属于 `USER32.DLL`；`USER32.DLL` 负责窗口、消息、用户界面相关功能，不是 `KERNEL32.DLL`。
- Windows API 调用里，`.lib` 导入库为 API 函数提供导入/重定位相关信息。
- 64 位 Windows 下通常不能直接运行 16 位 DOS 程序，要用 DOSBox 这类虚拟机/模拟环境。
- DOS 功能调用中，子功能号通常放 `AH`。
- 汇编语言本身不自带“丰富库函数”，能不能调用库/API 要看平台和链接环境。

## 3. 数据表示与存储

### 3.1 单位和范围

| 名称         | 大小   |
| ------------ | ------ |
| bit          | 1 位   |
| byte         | 8 位   |
| word         | 16 位  |
| dword / long | 32 位  |
| qword / quad | 64 位  |
| 双四字       | 128 位 |

主存储器通常是**字节编址**：一个地址对应一个字节。现代计算机中每个主存储单元通常存放 1 个 byte（8 位）。

范围：

```text
n 位无符号数：0 ~ 2^n - 1
n 位有符号补码：-2^(n-1) ~ 2^(n-1)-1
```

典型值：

- 8 位有符号：`-128 ~ 127`
- 16 位有符号：`-32768 ~ 32767`
- 32 位有符号：`-2^31 ~ 2^31-1`
- 64 位有符号：`-2^63 ~ 2^63-1`

### 3.2 GAS 数据伪指令

| C/含义        | 字节 | GAS                                 |
| ------------- | ---: | ----------------------------------- |
| `char`        |    1 | `.byte`                             |
| `short`       |    2 | `.word`                             |
| `int`         |    4 | `.long`                             |
| `long long`   |    8 | `.quad`（qword）                    |
| 指针 x86-64   |    8 | `.quad`（qword）                    |
| 字符串不补 0  |    n | `.ascii "abc"`                      |
| 字符串补 `\0` |  n+1 | `.asciz "abc"` / `.string "abc"`    |
| 分配空间      |    n | `.space n`，也可 `.space n, fill`   |
| 符号常量      |    0 | `.equ name,value` / `.set name,value` / `name = value` |

提醒：`long` 在不同平台可能 4 或 8 字节，考题若问 C 类型，要看题目平台和课件上下文。

`.equ` 定义的符号常量不可修改；`.set` 可重新赋值。`.space n, fill` 表示分配 n 字节并用 fill 填充，省略 fill 时通常填 0。

### 3.3 小端存储

x86-64 用小端：低位字节放低地址，高位字节放高地址。按字节存放

对比：大端方式是高位字节放低地址，低位字节放高地址。本课程主线 x86-64 按小端做题。

例：`.long 0x12345678`。十六进制，所以两个数是一字节

```text
低地址 -> 高地址：78 56 34 12
```

例：`.long 0x9ABCDEF0`

```text
低地址 -> 高地址：F0 DE BC 9A
```

list 文件题常用：

```asm
array: .word 1, 2 #定义了一个叫做array的数据标签，放了两个.word数据，一个两字节。
mov eax, dword ptr array[rip]  # 按 4 字节从 array 处取数
```

内存前 4 字节是 `01 00 02 00`，所以 `EAX = 0x00020001`。

### 3.4 补码和扩展

负数补码：

```text
绝对值取二进制 -> 按位取反 -> 加 1
```

符号扩展：位数变长但真值不变。

- 正数高位补 0。
- 负数高位补 1。
- 例：8 位补码 `0xF3` 扩展到 16 位为 `0xFFF3`。

常见指令：

- `movsx`：带符号扩展。
- `movzx`：零扩展。直接补0
- `cdq`：把 `EAX` 符号扩展到 `EDX:EAX`。
- `cqo`：把 `RAX` 符号扩展到 `RDX:RAX`。

背：无符号除法先清高半；有符号除法先符号扩展。

### 3.5 `.align` 与地址计算

`.align N`：把下一条数据/指令地址调整到 N 的倍数。

做题步骤：

1. 从当前偏移开始。
2. 每个 `.byte/.word/.long/.quad/.space` 按字节数累加。
3. 遇到 `.align N`，向上补到 N 的整数倍。
4. `.equ`、`name = expr` 不占空间。
5. list 文件里机器码按小端显示，要反过来组合多字节值。

例：当前地址 `0x000e`，执行 `.align 16`，下一地址是 `0x0010`。

关于list文件

汇编器生成的列表文件

比如当前 GAS 列表文件里常见：

```
地址      机器码        源程序
0000     B8 78 56 34 12      mov eax,0x12345678
0005     48 89 C3            mov rbx,rax
```

多字节立即数在机器码中按小端显示，例如 `0x12345678` 显示为 `78 56 34 12`。

- 每条汇编语句对应的偏移地址
- 生成的机器码/目标代码
- 原来的汇编源代码
- 有时还包括符号表、段信息、错误提示等

### 3.6 结构体在内存里怎么排布：对齐和填充

CPU 访问 int、指针这类数据时，通常希望它们的地址满足某种倍数关系，这叫 对齐要求。比如常见情况下：

```
char   对齐 1：可以放在任意地址
short  对齐 2：最好放在 2 的倍数地址
int    对齐 4：最好放在 4 的倍数地址
指针   对齐 8：64 位下通常放在 8 的倍数地址
```

结构体题虽然不一定大题考，但选择/判断容易出现。做法：

1. 每个成员按自己的对齐要求放置，常见 `char` 对齐 1、`short` 对齐 2、`int` 对齐 4、指针对齐 8。
2. 成员之间可能插入填充字节。
3. 结构体总大小通常补齐到最大成员对齐值的整数倍。
4. 数组元素连续存放，`A[i]` 地址 = 首地址 + `i * sizeof(A[0])`。

例：

```c
struct A {
    int a;
    char c;
    int b;
};
```

```
偏移 0：int a，占 4 字节
偏移 0,1,2,3 都属于 a

偏移 4：char c，占 1 字节
偏移 4 属于 c

接下来当前偏移是 5
但是 int b 要求按 4 字节对齐
5 不是 4 的倍数，所以不能直接放 b

于是补 3 个字节：
偏移 5,6,7 是填充字节

偏移 8：int b，占 4 字节
偏移 8,9,10,11 属于 b
```

常见 64 位环境下：`a` 偏移 0，`c` 偏移 4，后面填充 3 字节，`b` 偏移 8，`sizeof(struct A)=12`。

### 3.7 C 语言有符号/无符号混合比较

选择题里 C 语言比较也会绕人：

- **有符号数和无符号数比较时，有符号数通常先转换成无符号数**。
- 例：`int x=-1; unsigned int y=2; if (x>y)` 中，`x` 转成无符号后是很大的数，所以条件为真，输出 `-1`。
- 同一串二进制没有天然“正负”，关键看题目按有符号还是无符号解释。

## 4. 标识符、语句、常见语法坑

### 4.1 标识符

GAS 中合法标识符常见规则：

- 不能以数字开头。
- 不能是寄存器名、指令名、伪指令名。
- 标号后面加冒号。
- `.L1` 这种局部标号可以用。
- 十六进制数按 GAS 当前口径写 `0xFF`，不要写成其他汇编器风格。

常见判断：

- `again` 合法。
- `8var8` 不合法。
- `rax` 不合法。
- `mov` 不合法。
- `.data` 不合法。
- `.L1` 合法。

### 4.2 指令格式错误高频

1. 普通mov不能内存到内存。最多只能有一个内存操作数
2. 两个操作数大小必须一致
3. 立即数不能超过目标大小
4. 写内存立即数必须说明大小
5. 段寄存器不能直接接受立即数。段寄存器如 ds, es, ss, fs, gs 不能直接从立即数赋值。要先放进通用寄存器，再送入段寄存器：
6. 不能混用新增低 8 位和传统高 8 位（dil/sil/bpl/spl 不要和 ah/bh/ch/dh 混用）
7. 地址寄存器一般 64 位
8. xchg不能和立即数交换

普通双操作数指令通常要求：**不能两个都是内存，大小要一致，立即数要放得下，内存大小要明确，地址寄存器在 64 位模式下用 64 位寄存器。**

必背错误：

```asm
mov dvar[rip+4], dvar[rip]   # 错，普通 mov 不能内存到内存
mov edi, si                  # 错，操作数大小不一致
mov al, 0x50a                # 错，立即数超过 8 位
mov [rbx+4], 200             # 错，内存操作数类型不明确
mov ds, 100                  # 错，不能立即数直接送段寄存器
mov dil, ah                  # 错，不能混用新增低 8 位寄存器和 AH/BH/CH/DH
mov edx, [ebx+0xff]          # 64 位模式下错，基址/变址应使用 64 位寄存器
xchg 100, al                 # 错，xchg 不能用立即数
```

```
r 开头通常是 64 位：rax, rbx, rdi, rsi
e 开头通常是 32 位：eax, ebx, edi, esi
没有 r/e 的老名字通常是 16 位：ax, bx, si, di
以 l 结尾的一些是 8 位：al, bl, dil, sil
以 h 结尾的是高 8 位：ah, bh, ch, dh
```

正确习惯：

```asm
mov eax, dvar[rip]
mov dvar[rip+4], eax
mov dword ptr [rbx+4], 200
mov ax, 100
mov ds, ax
```

## 5. x86-64 运行环境和寻址

x86-64 选择题原词：

- x86-64 支持 64 位模式，64 位程序常按平展模式理解，程序员主要看到虚拟地址/线性地址，不直接控制物理地址。
- x86 课件口径常见 5 种工作方式：保护方式、实地址方式、系统管理方式、IA-32e 兼容模式、IA-32e 64 位模式。选择题若问“哪个不是 x86-64 处理器工作方式”，`IA-32 模式`容易作为干扰项。
- x86-64 支持实模式、段式/分段模式、平展模式等存储/地址组织口径；本届写 64 位程序时主要按平展模式和 RIP 相对寻址理解。
- 基本运行环境是程序员可见资源：通用寄存器、标志寄存器、内存、`RIP` 等；运算器是 CPU 内部部件，不算程序员直接使用的运行环境。

### 5.1 寄存器

| 64 位  | 低 32 位 | 低 16 位 | 低 8 位  |
| ------ | -------- | -------- | -------- |
| RAX    | EAX      | AX       | AL/AH    |
| RBX    | EBX      | BX       | BL/BH    |
| RCX    | ECX      | CX       | CL/CH    |
| RDX    | EDX      | DX       | DL/DH    |
| RSI    | ESI      | SI       | SIL      |
| RDI    | EDI      | DI       | DIL      |
| RBP    | EBP      | BP       | BPL      |
| RSP    | ESP      | SP       | SPL      |
| R8-R15 | R8D-R15D | R8W-R15W | R8B-R15B |

注意：

- `RAX=0x123456789ABCDEF0` 时，`EAX=0x9ABCDEF0`。
- 写入 32 位寄存器通常会清零高 32 位。
- `SI` 是 16 位，`SIL` 才是 8 位。

### 5.1.1 寄存器用途速查

| 寄存器             | 文章中主要用途                         | 记忆点                         |
| ------------------ | -------------------------------------- | ------------------------------ |
| `RAX/EAX/AX/AL/AH` | 返回值、累加运算、乘除法、I/O 临时数据 | 最常见的结果寄存器             |
| `RBX/EBX/BX/BL/BH` | 普通数据、除数、基址寄存器             | 可放数据，也常参与地址         |
| `RCX/ECX/CX/CL/CH` | 循环计数、参数传递、位移次数           | `loop` 默认用 `ECX/RCX`        |
| `RDX/EDX/DX/DL/DH` | 乘除法高位、余数、参数传递             | 除法后余数在 `EDX/RDX`         |
| `RSI/ESI/SI/SIL`   | 源地址、数组/字符串指针、Linux 参数    | `SI` 可记 source               |
| `RDI/EDI/DI/DIL`   | 目的地址、数组/字符串指针、Linux 参数  | `DI` 可记 destination          |
| `RBP/EBP/BP/BPL`   | 栈帧基址                               | 常用于访问栈中参数/局部变量    |
| `RSP/ESP/SP/SPL`   | 栈顶指针                               | `push/pop/call/ret` 都会影响它 |
| `R8-R15`           | x86-64 新增通用寄存器                  | 可传参、可存数据、可寻址       |

最短背诵：

```text
A 放结果，D 放余数，C 管循环，SP 管栈，BP 管栈帧，SI/DI 管地址，FLAGS 管判断。
```

乘除法固定搭配：

| 位宽  | 被除数    | 除数  | 商    | 余数  |
| ----- | --------- | ----- | ----- | ----- |
| 16 位 | `DX:AX`   | r/m16 | `AX`  | `DX`  |
| 32 位 | `EDX:EAX` | r/m32 | `EAX` | `EDX` |
| 64 位 | `RDX:RAX` | r/m64 | `RAX` | `RDX` |

记忆：**商在 A，余在 D**。

循环、栈、调用约定常考：

- `loop label` 默认使用 `ECX/RCX`，自动减 1；`loop` 不影响标志位。
- `RSP` 指向栈顶，64 位下 `push` 使 `RSP -= 8`，`pop` 使 `RSP += 8`。
- `call` 把返回地址压栈，再跳到目标；`ret` 弹出返回地址，64 位下使 `RSP += 8`。
- Win64 前 4 个整数参数常用 `RCX/RDX/R8/R9`，返回值在 `RAX/EAX`。
- x86-64 Linux 前 6 个整数参数常用 `RDI/RSI/RDX/RCX/R8/R9`，返回值在 `RAX/EAX`。

其他容易混的寄存器：

- `RIP`：当前指令位置，常见 `symbol[rip]` 这种 RIP 相对寻址。
- `RFLAGS`：标志寄存器，保存 `CF/OF/SF/ZF/PF` 等标志。
- 段寄存器如 `DS/ES/SS/FS/GS` 不能直接接受立即数，通常要先放入通用寄存器再送入段寄存器。
- `AH/BH/CH/DH` 不能和 `SIL/DIL/BPL/SPL/R8B...` 在同一条需要 REX 的指令里混用。

### 5.2 地址类型

- 64 位程序员通常看到的是虚拟地址/线性地址，不直接控制物理地址。
- 当前 GAS x86-64 代码访问全局变量时，常见 `label[rip]` 这种 RIP 相对寻址。
- 地址寄存器和变址寄存器在 64 位模式下一般使用 64 位寄存器，如 `rax/rbx/rcx/rdx/rsi/rdi/r8-r15`。

### 5.3 寻址方式

内存寻址通式：

```asm
[base（基址寄存器） + index（变址寄存器） * scale（比例因子，只能说1，2，，4，8） + displacement（偏移量）]
```

有效地址：

```text
EA = 基址寄存器 + 变址寄存器 * 比例因子 + 位移量
```

常见分类：

| 例子                  | 寻址方式           |
| --------------------- | ------------------ |
| `mov eax, 10`         | 立即数寻址         |
| `mov eax, ebx`        | 寄存器寻址         |
| `mov eax, [rbx]`      | 寄存器间接寻址     |
| `mov eax, [rbx+4]`    | 寄存器相对寻址     |
| `mov eax, [rcx+r8*4]` | 带比例基址变址寻址 |
| `mov eax, x[rip]`     | RIP 相对寻址       |

数组公式：

```text
int A[i] 地址 = A 首地址 + i * 4
char S[i] 地址 = S 首地址 + i
long long B[i] 地址 = B 首地址 + i * 8
```

`lea` 重点：

- `lea` 只计算地址表达式，不访问内存。
- 常用于算术表达式。

例：

```asm
lea rdx, [rax+rax*8+7]    # rdx = 9*rax + 7
lea eax, [rcx+rdx*4]      # eax = x + 4*y
```

## 6. 标志位与运算

### 6.1 标志位

| 标志 | 含义          | 判断                         |
| ---- | ------------- | ---------------------------- |
| CF   | Carry Flag    | 无符号加法进位 / 减法借位    |
| OF   | Overflow Flag | 有符号运算溢出               |
| SF   | Sign Flag     | 结果最高位为 1               |
| ZF   | Zero Flag     | 结果为 0                     |
| PF   | Parity Flag   | 结果低 8 位中 1 的个数为偶数 |

### 6.2 加减法判标志

做题步骤：

1. 明确位宽：8/16/32/64。
2. 按无符号完整计算。
3. 截断到当前位宽。
4. CF：完整结果是否超出位宽；减法是否借位。
5. OF：有符号是否溢出。
6. SF：截断结果最高位。
7. ZF：截断结果是否 0。
8. PF：截断结果低 8 位中 1 的个数是否为偶数。

OF 口诀：

- 加法：同号相加结果变号，OF=1。
- 减法：异号相减结果变号，OF=1。

例：

```text
0xB7 + 0x7E = 0x135 -> 8 位结果 0x35
CF=1, OF=0, SF=0, ZF=0, PF=1
```

### 6.3 逻辑指令

| 指令   | 作用                           |
| ------ | ------------------------------ |
| `and`  | 某些位清 0，保留其他位         |
| `or`   | 某些位置 1，保留其他位         |
| `xor`  | 某些位取反，保留其他位         |
| `test` | 只做与运算设置标志，不保存结果 |
| `not`  | 按位取反，不影响标志           |

口诀：

```text
AND：1 & x = x，0 & x = 0
OR ：1 | x = 1，0 | x = x
XOR：1 ^ x = ~x，0 ^ x = x
```

`and/or/xor/test` 执行后通常：

- `CF=0`
- `OF=0`
- 按结果更新 `SF/ZF/PF`

例：

```asm
xor eax, eax    # EAX=0，CF=0，OF=0，ZF=1，SF=0，PF=1
and al, 0x80    # AL 低 7 位肯定为 0
xor eax, 0x8001 # EAX 的 D15 和 D0 取反
```

### 6.4 `test`、`neg`、标志保存

`test a,b`：只做按位**与**并设置标志，不保存结果。常用来判断 0 或判断某一位。

```asm
test eax, eax
jz zero          # eax == 0
js negative      # eax 最高位为 1

test al, 0x80
jnz high_bit_1   # AL 最高位为 1
```

`neg x`：求相反数，等价于 `x = 0 - x`。常用于绝对值：

```asm
mov eax, ecx
test eax, eax
jns done
neg eax
done:
```

`neg` 本质是0-x，对标志位有影响：操作数原来为 0 时 `CF=0`，否则 `CF=1`；结果也会设置 `ZF/SF/OF/PF` 等。

保存/恢复标志：

```asm
pushfq   把当前RFLAGS（就是当前各种标志位）压入栈中
pop rax       # 读 RFLAGS 到 RAX

push rax
popfq         # 用 RAX 恢复 RFLAGS
```

## 7. 移位、循环移位、乘除法

### 7.1 移位

| 指令          | 含义                   | 移出位      |
| ------------- | ---------------------- | ----------- |
| `shl/sal x,1` | 左移，低位补 0         | 最高位进 CF |
| `shr x,1`     | 逻辑右移，高位补 0     | 最低位进 CF |
| `sar x,1`     | 算术右移，高位补符号位 | 最低位进 CF |

用途：

- `shl n` 约等于无符号乘 `2^n`。
- `shr n` 约等于无符号除 `2^n`。
- `sar n` 约等于有符号除 `2^n`，但负数且有余数时和 `idiv` 的截断方向可能不同。

例：

```asm
mov ecx, eax
shl eax, 3
lea eax, [rax+rax*8]
sub eax, ecx
```

计算：

```text
原 eax = x
shl eax,3 -> 8x
lea eax,[rax+rax*8] -> 8x + 64x = 72x
sub eax,ecx -> 71x
```

一条乘法指令：

```asm
imul eax, 71
```

### 7.2 乘除法

常见：

```asm
imul eax, 9      # eax = eax * 9
mul r/m32  （表示操作数可以是32位内存操作数或者寄存器）      # 无符号乘法
imul r/m32       # 有符号乘法
div r/m32        # 无符号除法
idiv r/m32       # 有符号除法
```

除法不能简单的算

例如：

```
被除数 = EDX:EAX   ; EDX 是高 32 位，EAX 是低 32 位，合起来 64 位
除数   = EBX
商     -> EAX
余数   -> EDX
```

除法寄存器搭配：商后余前

| 位宽      | 被除数    | 除数  | 商    | 余数  |
| --------- | --------- | ----- | ----- | ----- |
| 16 位除法 | `DX:AX`   | r/m16 | `AX`  | `DX`  |
| 32 位除法 | `EDX:EAX` | r/m32 | `EAX` | `EDX` |
| 64 位除法 | `RDX:RAX` | r/m64 | `RAX` | `RDX` |

准备被除数：

```asm
# 无符号 32 位
xor edx, edx
div ebx（只要一个操作数，EDX：EAX是被除数规定好了，商EAX，余数EDX）

# 有符号 32 位
cdq（Convert Doubleword to Quadword符号位扩展，已经隐含了操作数。cdq是把 EAX 的符号位扩展到 EDX）
idiv ebx

# 有符号 64 位
cqo（Convert Quadword to Octword把 RAX 的符号位扩展到 RDX）
idiv rbx
```

混合四则模板：

```asm

# W = (X * Y - V) / Z
# eax = X, ebx = Y, ecx = V, edx = Z, rsi = &W
# [rsi] 保存商，[rsi+4] 保存余数

mov edi, edx              # 先保存除数 Z，因为 cdq 会修改 edx

test edi, edi             # 判断除数是否为 0
jz div_zero               # 如果 Z == 0，跳转处理除零

imul eax, ebx             # eax = X * Y
sub eax, ecx              # eax = X * Y - V

cdq                       # 将 eax 符号扩展到 edx:eax
idiv edi                  # edx:eax / edi
                           # eax = 商，edx = 余数

mov dword ptr [rsi], eax      # 保存商到 W
mov dword ptr [rsi+4], edx    # 保存余数到 W+4

```

### 7.3 128 位大数加减

**128 位数拆成两个 64 位数算，先算低 64 位，再把进位/借位传给高 64 位。**

**加法：**
低位 add，产生进位 CF
高位 adc，把 CF 加进去

**减法：**
低位 sub，产生借位 CF
高位 sbb，把 CF 减进去

双 64 位表示 128 位：

```text
A = A_hi:A_lo
B = B_hi:B_lo
```

加法：

**adc是add with carry，带进位加法，`rdx = A_hi + B_hi + CF`**

```asm
add rax, rbx    # low = A_lo + B_lo，设置 CF
adc rdx, rcx    # high = A_hi + B_hi + CF
```

减法：

```asm
sub rax, rbx    # low = A_lo - B_lo，设置 CF(借位)
sbb rdx, rcx    # high = A_hi - B_hi - CF
```

背：低位用 `add/sub`，高位用 `adc/sbb` 衔接 CF。

### 7.4 循环移位和双精度移位

循环移位常考“移出去的位到哪里”：

**s是普通移位，r是循环移位**

| 指令      | 含义                                                  |
| --------- | ----------------------------------------------------- |
| `rol x,n` | 不带 CF 循环左移，最高位绕回最低位，最后移出的位进 CF |
| `ror x,n` | 不带 CF 循环右移，最低位绕回最高位，最后移出的位进 CF |
| `rcl x,n` | 带 CF 循环左移，把 CF 也当作循环链的一位              |
| `rcr x,n` | 带 CF 循环右移，把 CF 也当作循环链的一位              |

双精度移位：

```asm
shld dest, src, n   # dest 左移 n 位，空出来的低位由 src 的高位补入。类似拼接成dest:src
shrd dest, src, n   # dest 右移 n 位，空出来的高位由 src 的低位补入类似拼接成src:dest
```

用途：多字数据移位、跨寄存器拼接。判断时先画 `src:dest` 的位串，再看目标寄存器如何变化。

## 8. 条件跳转、分支、循环

### 8.1 `cmp`

```asm
cmp a, b
```

只做 `a - b` 设置标志，不保存结果。

可以把 `cmp a,b` 理解成一次“假减法”：

```asm
cmp eax, ebx     # 偷偷算 eax-ebx，只改标志位，eax/ebx 都不变
je  equal        # ZF=1，说明 eax-ebx=0，即 eax==ebx
ja  above        # 无符号看，eax > ebx
jg  greater      # 有符号看，eax > ebx
```

`cmp a,b` 后常用标志：

| 情况            | 标志现象        | 含义                   |
| --------------- | --------------- | ---------------------- |
| `a == b`        | `ZF=1`          | 减完为 0               |
| 无符号 `a < b`  | `CF=1`          | 减法需要借位           |
| 无符号 `a >= b` | `CF=0`          | 减法不借位             |
| 有符号比较      | 看 `SF` 和 `OF` | 结果符号要结合溢出修正 |

核心：CPU（Central Processing Unit，中央处理器）不知道你想按有符号还是无符号解释，后面的跳转指令决定解释方式。

容易错的例子：

```asm
mov al, 0ffh      # AL = 255，也可解释为 -1
cmp al, 1         # 只算 0ffh-1，AL 仍是 0ffh
ja  unsigned_big  # 会跳：按无符号 255 > 1
jl  signed_small  # 也会跳：按有符号 -1 < 1
```

所以：`cmp` 只负责产生标志；`ja/jb` 按无符号解释，`jg/jl` 按有符号解释。

### 8.1.1 只设置标志、不保存结果的常见指令

| 指令       | 偷偷做了什么       | 常用场景              | 是否保存结果 |
| ---------- | ------------------ | --------------------- | ------------ |
| `cmp a,b`  | 算 `a-b`           | 比较大小、配合 `jcc`  | 不保存       |
| `test a,b` | 算 `a & b`         | 判断 0、判断某一位    | 不保存       |
| `bt a,b`   | 取第 `b` 位到 `CF` | 判断某一位是 0 还是 1 | 不保存       |

最常考的是 `cmp` 和 `test`：

```asm
cmp eax, 10
jl  less_than_10     # 比较 eax 和 10

test eax, eax
jz  is_zero          # 判断 eax 是否为 0
js  is_negative      # 判断 eax 最高位是否为 1

test al, 00000001b
jnz odd              # 判断最低位是否为 1，即奇数

bt eax, 5
jc bit5_is_1         # 判断 EAX 的第 5 位是否为 1
```

对比记忆：

- `cmp` 像 `sub`，但不保存减法结果。
- `test` 像 `and`，但不保存按位与结果。
- `bt` 只把某一位送进 `CF`，常配合 `jc/jnc`。
- `add/sub/and/or/xor/inc/dec/neg/shl/shr` 等会设置标志，但它们也会修改操作数，不属于“只设置标志不保存结果”。

### 8.2 无符号跳转

| 指令          | 条件         | C 语言 |
| ------------- | ------------ | ------ |
| `ja`          | CF=0 且 ZF=0 | `>`    |
| `jae` / `jnb` | CF=0         | `>=`   |
| `jb` / `jc`   | CF=1         | `<`    |
| `jbe`         | CF=1 或 ZF=1 | `<=`   |

口诀：无符号看 `above/below`，主要看 CF/ZF。

### 8.3 有符号跳转

| 指令          | 条件           | C 语言 |
| ------------- | -------------- | ------ |
| `jg` / `jnle` | ZF=0 且 SF=OF  | `>`    |
| `jge` / `jnl` | SF=OF          | `>=`   |
| `jl` / `jnge` | SF!=OF         | `<`    |
| `jle` / `jng` | ZF=1 或 SF!=OF | `<=`   |

口诀：有符号看 `greater/less`，要看 SF 和 OF。

### 8.4 其他常用跳转

| 指令      | 含义                   |
| --------- | ---------------------- |
| `je/jz`   | 等于 / ZF=1            |
| `jne/jnz` | 不等 / ZF=0            |
| `jc`      | CF=1                   |
| `jnc`     | CF=0                   |
| `js`      | SF=1                   |
| `jns`     | SF=0                   |
| `jmp`     | 无条件转移，不影响标志 |

短转移：机器码常见 `EB xx`，位移是 8 位有符号数，约 `-128 ~ +127`。

短转移 EB xx：xx 是相对位移，不是绝对地址；从下一条指令算起，范围约为前 128 字节到后 127 字节。

转移目标也会考概念：

- `jmp again`：直接转移，目标来自标号 `again`。
- `jmp rbx` / `jmp qword ptr [rdi]`：间接转移，目标来自寄存器或内存。
- `short/near/far` 描述转移距离或段内段间属性；`byte/dword` 描述数据大小，不是标号本身包含的转移属性。
- `jecxz label` 看 `ECX==0`，`jrcxz label` 看 `RCX==0`；64 位题如果强调 `RCX`，优先写 `jrcxz`。

### 8.5 循环结构：

循环三部分：

1. 循环初值。
2. 循环体。
3. 循环控制。

**loop 是一种“用 ECX/RCX 当计数器的自动减 1 循环跳转指令**

可以理解为

```
ecx = ecx - 1
if ecx != 0:
    jump label
```

`loop label`：

```asm
loop label
```

等价功能近似：

```asm
dec ecx # dec是减1
jnz label #ZF不为0就跳转label
```

但重要区别：`loop` 不影响标志，`dec` 会影响除 CF 外的多个标志。

课件还列出两类带 ZF 条件的循环，按选择题记：

```asm
loope label    # loopz：计数减 1 后，计数不为 0 且 ZF=1 才跳
loopne label   # loopnz：计数减 1 后，计数不为 0 且 ZF=0 才跳
```

`loope` 和 `loopz` 同义；`loopne` 和 `loopnz` 同义。它们和 `loop` 一样会先减计数寄存器，目标地址通常是相对短转移。

```
cmp eax, ebx
loop L1
je equal
```

这里 loop 不会破坏前面 cmp 设置的标志位，所以后面的 je 仍然看的是 cmp eax, ebx 的结果。如果你把 loop 换成：

```
dec ecx
jnz L1
je equal
```

那就不一样了，因为 dec ecx 会改 ZF 等标志，je 看的就可能是 dec ecx 的结果，而不是 cmp 的结果。

陷阱：若 `ECX=0` 后执行 `loop`，会先减为 `0xFFFFFFFF`，然后发生转移，不是循环 0 次。

因为loop是

```
ECX = ECX - 1
if ECX != 0 jump
```

所以会进入一个超大的循环

所以如果你想表达“ECX 为 0 就别进循环”，通常要先判断：

```
jecxz done # 如果ecx为0直接跳转到done
L:
    ; body
    loop L
done:
```

jecxz label 的意思是：

```
if ecx == 0:
    jump label
```

`jecxz label`：若 `ECX=0` 就跳转，不先减 1。选择题常把它和 `loop` 放一起问：`ECX=0` 时，`loop label` 会转移，`jecxz label` 也会转移，但原因不同。

`loop` 常用于减量计数循环，不是增量计数循环。

数组遍历模板：

```asm
lea rcx, array[rip]      # rcx = 数组首地址
xor eax, eax             # eax = 0，也就是 i = 0。

.L:
    cmp eax, n           # 比较 i 和 n
    jge .done            # 如果 i >= n，结束

    mov edx, dword ptr [rcx+rax*4]
                          # edx = array[i]
                          # 因为每个 int 是 4 字节，所以偏移是 i*4。地址通常是 64 位地址，所以基址寄存器和变址寄存器一般用 64 位寄存器：

    # body                # 对 array[i] 做操作

    inc eax              # i++
    jmp .L               # 回到循环开头

.done:
```

tip: `symbol[rip]`都表示通过 RIP 相对寻址找到 symbol 的地址附近的内存位置。

## 9. 栈、CALL、RET、子程序

### 9.1 栈操作

x86-64 栈向低地址增长，`RSP` 指向栈顶。

```asm
push rax
```

等价：

```text
RSP = RSP - 8
[RSP] = RAX
```

```asm
pop rdx
```

等价：

```text
RDX = [RSP]
RSP = RSP + 8
```

ret：函数返回指令。从栈顶取出一个返回地址，然后跳转到那个地址执行

`pop rip`

相当于

```
RIP = [RSP]   ; 从栈顶取出返回地址
RSP += 8      ; 64 位下弹出 8 字节
```

`rip`:64 位 CPU 里的指令指针寄存器。

`ret n`：ret n 的意思是：先执行普通 ret，再额外把栈顶指针加上 n 字节。可以理解成：

```
RIP = [RSP]
RSP = RSP + 8 + n
```

```text
ret + add rsp,n
```

### 9.2 CALL：先记住回来位置，再跳去函数

`call target` 两件事：

1. 把 call 后面那条指令的地址压入栈中。这个地址叫“返回地址”。
2. 把目标地址送入 RIP。也就是跳到 func 开始执行。

```
jmp  只跳走，不记返回地址
call 跳走之前，会把返回地址压栈
```

函数执行完后，一般会执行`ret`, 从栈顶弹出刚刚保存的返回地址，再跳回去。

在汇编代码里面，我们写的target一般就是目标标签/地址

```
call func
call 0x401000
```

但是在机器码里面，一般传入的是相对位移

相对近调用目标：很多 call 指令不是直接写完整目标地址，而是写一个“相对位移”。格式类似`E8 xx xx xx xx`

- E8是call rel32的机器码
- 后面四字节是有符号位移
- 目标地址不是直接等于位移，而是

```text
目标地址 = call 下一条指令地址 + 有符号位移
```

**注意基准是“下一条指令地址”，不是 call 自己的地址。**

实际上就是说

```
你写 call target，汇编器帮你把 target 转成相对位移；
CPU 执行时用“下一条指令地址 + 位移”算出真正目标地址。
```

例：`call` 在 `0x...171e`，机器码 5 字节，下一条是 `0x...1723`，位移 `0xFFFFFFED=-19`，目标 `0x...1710`。

### 9.3 子程序结构：汇编里的“子程序”就是高级语言里的函数

call sub 相当于调用函数，ret 相当于函数返回。

```asm
.text
.globl main
main:
    # 准备参数
    call sub
    # 输出/收尾
    mov eax, 0
    ret

sub:
    # 子程序体
    ret
```

执行流程是：

1. 程序从 `main` 开始。
2. 执行到 `call sub`，CPU 跳到 `sub:` 标号处。
3. `sub` 执行到 `ret`，返回到 `call sub` 的下一条指令。
4. 继续执行 `mov eax, 0` 和 `ret`。

对应高级语言大概就是：

```c
int main(void)
{
    sub();
    return 0;
}

void sub(void)
{
}
```

`call` 不只是普通跳转，它会自动把“返回地址”压入栈中；`ret` 再从栈里取出返回地址并跳回去。

```text
call sub  约等于：push 返回地址 + jmp sub
ret       约等于：pop 返回地址 + jmp 返回地址
```

考试抓关键词：调用子程序用 `call`，子程序返回用 `ret`，返回地址通常保存在栈中。

课件要求掌握三种参数传递方式：

| 方式     | 做法                                     | 记忆点                                   |
| -------- | ---------------------------------------- | ---------------------------------------- |
| 寄存器   | 参数放入约定寄存器，子程序从寄存器取     | 最简单常用，少量数据传值，大量数据传地址 |
| 共享变量 | 主程序和子程序访问同一个全局变量/数据标号 | 模块间传数据方便，但子程序通用性较差     |
| 堆栈     | 主程序 `push` 参数，子程序按栈帧取参数   | 普遍但复杂，调用后注意平衡堆栈           |

入口参数是主程序传给子程序的数据；出口参数是子程序返回给主程序的数据。带出口参数的寄存器通常不能保护后恢复，否则会把返回结果恢复没了。

当前 x86-64 子程序参数优先按 ABI 使用寄存器传递：**调用前把参数放入约定好的寄存器，子程序直接用寄存器取参数**。

```asm
mov edi, 3
mov esi, 5
call add
```

#### ABI（Application Binary Interface，应用程序二进制接口）。

不同平台的 ABI 不一样，所以“函数参数放哪里”不一样

- Win64 常见前 4 个整数参数：`RCX/RDX/R8/R9`，返回值 `RAX/EAX`，调用 API（Application Programming Interface，应用程序编程接口）要影子空间和栈 16 字节对齐。
- x86-64 Linux ABI 前 6 个整数参数：`RDI/RSI/RDX/RCX/R8/R9`，返回值 `RAX/EAX`。

ABI 可以理解成“函数调用规矩”。同样都是 `call func`，不同平台会约定：

- 前几个参数放在哪些寄存器。
- 参数太多时，多出来的参数怎么放到栈里。
- 返回值放在哪个寄存器。
- 哪些寄存器由调用者保存，哪些由被调用者保存。
- 调用前栈要不要 16 字节对齐。

所以写子程序或分析子程序时，不能只看 `call`，还要看题目给的是 Win64 还是 x86-64 Linux。题面出现 `rdi/rsi` 多半是 Linux x86-64；出现 `rcx/rdx/r8/r9`、Windows API 或 shadow space 多半按 Win64。

返回值重点记：整数返回值通常放在 `RAX/EAX`。例如 `int` 返回值看 `EAX`，64 位整数或地址返回值看 `RAX`。

Win64 调用四参数函数大概是：

```asm
mov rcx, a
mov rdx, b
mov r8,  c
mov r9,  d
call func
```

也就是类似：

```c
func(a, b, c, d);
```

Win64 记忆顺序：`RCX, RDX, R8, R9`。  
调用 Windows API 时还常考两个词：

- 影子空间：调用者通常要在栈上预留 32 字节，给被调用函数保存前 4 个寄存器参数用。
- 栈 16 字节对齐：调用前后要按 ABI 要求保持栈对齐，否则调用库函数/API 可能出问题。

考前背到这个程度一般够用：Win64 前 4 个整数/指针参数走 `RCX/RDX/R8/R9`，返回走 `RAX/EAX`，调 API 注意影子空间和 16 字节对齐。

x86-64 Linux 调用六参数函数大概是：

```asm
mov rdi, a
mov rsi, b
mov rdx, c
mov rcx, d
mov r8,  e
mov r9,  f
call func
```

也就是类似：

```c
func(a, b, c, d, e, f);
```

x86-64 Linux 记忆顺序：`RDI, RSI, RDX, RCX, R8, R9`。  
如果是 32 位 `int` 参数，常看这些寄存器的低 32 位形式：

```text
EDI, ESI, EDX, ECX, R8D, R9D
```

例：Linux x86-64 下写 `int add(int a, int b)`，第一个参数 `a` 在 `EDI`，第二个参数 `b` 在 `ESI`，返回值放 `EAX`：

```asm
add:
    mov eax, edi
    add eax, esi
    ret
```

对应 C 代码：

```c
int add(int a, int b)
{
    return a + b;
}
```

这个是本届 64 位题更应该优先掌握的子程序写法。

保护寄存器：子程序一般只保护约定要求保持不变、且自己会改动的寄存器；入口参数寄存器和出口返回寄存器通常不属于“必须保护后原样恢复”的对象，否则返回值/参数处理反而被破坏。

判断题口径：子程序可以写在代码段中 `main` 标号前，也可以写在 `main` 后；能被调用到即可。子程序定义通常以子程序名标号开始，以 `ret` 返回。

一句话背：子程序是一段可以被 `call` 调用、执行完用 `ret` 返回的代码；参数怎么传、返回值放哪，要看 ABI。Linux x86-64 背 `RDI/RSI/RDX/RCX/R8/R9`，Win64 背 `RCX/RDX/R8/R9`。

## 10. 多模块、C 混合编程、宏

### 10.1 多模块方法

一个汇编程序不一定写在一个源程序里面，可以多种组织方式

1. 源文件包含：`.include "sub.s"`，相当于把文件内容插进来。类似c语言`#include`。是简单粗暴的文本插入。相当于直接复制插入到这个位置，最后当作一个源程序汇编。因此包含的文件不一定有完整的框架。
2. 模块连接--真正的多文件：各文件分别汇编/编译成 `.o`，链接成一个可执行文件。

```
# main.s
.globl main
main:
    call func
    ret
```

```
# sub.s
.globl func
func:
    ret
```

汇编成两个目标文件

```
main.s -> main.o
sub.s  -> sub.o
```

然后再链接：`main.o + sub.o -> 可执行文件`

模块指的是.o这种目标文件

`.globl func`：把func公开，让别的目标文件也能看见

`.extern other_func`告诉汇编器这个函数在别的模块（但也可以不写，最后链接的时候会去其他模块找）

3. 子程序库：多个 `.o` 打包成 `.a`，链接时使用。

GAS 常用：

```asm
.globl func
func:
    ret
```

外部符号：

```asm
.extern other_func     # 可写，可不写；GAS 通常允许直接引用外部符号
.globl my_func         # 本模块定义并导出 my_func
```

模块连接注意：

- 模块是目标文件 `.o`，不是源文本。
- 被 C 调用的汇编模块不需要再写 `main`，写要导出的函数即可。
- 模块连接方式下，每个独立汇编源文件通常要符合源程序框架。
- 源文件包含方式下，被 `.include` 包含的片段不一定完整，不要求每个片段都有完整框架。
- 子程序库是多个二进制目标模块的集合，不要求模块由同一种编程语言写成。

### 10.2 C 和 GAS 混合

形式：

- 模块连接。c文件和汇编分开写，最后一起链接

例如：

```c
// main.c
extern int add(int a, int b);

int main() {
    return add(1, 2);
}
```

```asm
# add.s, GAS Intel 语法，Linux x86-64 ABI
.intel_syntax noprefix
.text
.globl add
add:
    mov eax, edi
    add eax, esi
    ret
```

分别编译再链接：`gcc main.c add.s -o main`

用 gcc 把 C 文件 main.c 和 汇编文件 add.s 一起编译、汇编、链接，最后生成一个可执行文件，名字叫 main。


- 嵌入汇编 / 扩展 `asm`。

最基本：`asm("nop")`。它只能单纯插入汇编指令，不适合和 C 变量可靠交互。

扩展asm。告诉编译器三件事：执行什么汇编，汇编结果放到那个c变量，要用哪些c变量

```c
asm volatile(
    "汇编模板"
    : 输出操作数
    : 输入操作数
    : 破坏列表
);
```

```c
asm volatile(
    "add %0, %1"//汇编模板，按 Intel 语法理解为 a = a + b
    : "+r"(a) // 是说a 要放进一个寄存器。r表示通用寄存器，+表示这个变量是输入又是输出。意思是请编译器找个寄存器放 a，汇编执行完后，再把结果更新回 C 变量 a。
    : "r"(b)//请编译器找个寄存器放 b，汇编里面会用到它，但不会修改它。
    : "cc"//"cc" 表示这段汇编会改变条件码，也就是标志位。
);
```


扩展 asm 高频：

- `asm volatile(...)`：禁止编译器优化/删除这段汇编。
- 破坏内存写 `"memory"`。
- 破坏条件码写 `"cc"`。
- 基本 asm 不能可靠直接和 C 变量交互；扩展 asm 才能声明输入输出约束。

### 10.3 GAS 宏

宏不是“运行时跳过去执行的一段代码”，而是“汇编前/汇编时把代码复制粘贴到调用位置”，这个过程叫**宏展开**。

```asm
.macro LOGICAL op, a, b
    mov eax, \a
    \op eax, \b
.endm
```

它的意思是定义一个宏，名字叫 LOGICAL，有 3 个参数：

这里的 \a、\b、\op 表示“使用宏参数”。

之后写`LOGICAL and, ebx, ecx`，会展开成

```
mov eax, ebx
and eax, ecx
```


背：

- 宏通常先定义后调用。
- 宏调用不使用 `call`，不需要 `ret`。
- 宏定义不必须放在 `.text`。
- GAS 宏开头是 `.macro 宏名 参数列表`，结尾是 `.endm`。

| 对比 | 宏 | 子程序 |
|---|---|---|
| 发生时间 | 汇编时展开 | 运行时调用 |
| 本质 | 源代码替换 | 程序跳转执行 |
| 调用方式 | 直接写宏名 | `call 子程序名` |
| 是否需要 `ret` | 不需要 | 通常需要 |
| 代码体积 | 每调用一次展开一份，可能变大 | 只有一份代码 |
| 执行开销 | 没有 `call/ret` 开销 | 有调用返回开销 |

## 11. 分析题做法

### 11.1 C 语言转汇编

步骤：

1. 标出参数所在寄存器。
2. 标出返回值寄存器 `eax/rax`。
3. 普通赋值用 `mov`。
4. 地址计算优先用 `lea`。
5. 数组用 `[base+index*size]`。
6. `if` 用 `cmp/test + jcc`。
7. `for/while` 写初值、判断、循环体、更新、跳回。

例：`int add(int x, int y, int *result)`，题目给 `x=ecx, y=edx, result=r8, 返回 eax`：

```asm
add:
    mov eax, 0
    add ecx, edx
    jo .done 
    mov dword ptr [r8], ecx
    mov eax, 1
.done:
    ret
```

如果没有有符号溢出，就把和写到 *result，并返回 1；如果发生溢出，就不写结果，返回 0。

### 11.2 汇编推 C 语言

步骤：

1. 看函数入口寄存器，给变量命名。
2. 逐句写寄存器含义。
3. `cmp/test + jcc` 还原成 `if/while/for`。
4. `[base+index*scale]` 还原数组/指针。
5. `eax/rax` 是返回值。

例：

```asm
sub rsi, rdx
imul rdi, rsi
mov rax, rsi
sal rax, 63
sar rax, 63
xor rax, rdi
```

若 `rdi=x, rsi=y, rdx=z`：

```c
long long t1 = y - z;
long long t2 = x * t1;
long long mask = (t1 << 63) >> 63; // t1最低位为1则全1，否则全0
return t2 ^ mask;
```

### 11.3 list 列表文件题

必做步骤：

1. 先写每个变量起始偏移。
2. 数据伪指令换字节数。
3. `.align` 调整地址。
4. `.equ` 不占空间。
5. 机器码按小端解释。
6. 最后再回答寄存器值。

当前 GAS/x86-64 示范：

```asm
array: .word 1, 2
dvar:  .long 0x9abcdef0
```

`mov eax, dword ptr array[rip]`：取 `array` 前两个 `.word`，内存字节为 `01 00 02 00`，小端得 `0x00020001`。

### 11.4 条件跳转题

做法：

1. `cmp a,b` 先算 `a-b`。
2. 写出 CF/OF/SF/ZF。
3. 看跳转指令是有符号还是无符号。
4. 判断是否跳过后续赋值。

例：

```asm
mov eax, 8000003fh
mov edx, 8000003ah
cmp edx, eax
jl L1
mov eax, edx
L1:
```

`edx-eax = -5`，所以 `CF=1, OF=0, SF=1, ZF=0`。按有符号，`edx < eax`，`jl` 跳转，`eax` 保持 `0x8000003f`。

若改 `jb`，它看无符号 `CF=1`，本例也跳，结果相同，但理由不同。

## 12. 编程题模板

### 12.1 完整程序骨架

```asm
.intel_syntax noprefix # 用GAS的Intel语法

.data # 数据段
msg: .asciz "result:\n"

.text # 代码段
.globl main #把main暴露给连接器
main:
    # 初始化/准备参数
    call sub
    # 输出/收尾
    mov eax, 0
    ret

sub:
    # 子程序体
    ret
```

考试别只写核心几行。至少要有：

课程常用输入输出子程序名要能看懂：

| 子程序    | 常见功能                                                 |
| --------- | -------------------------------------------------------- |
| `dispmsg` | 显示以 0 结尾的字符串，常用 `rax` 传字符串地址           |
| `dispc`   | 显示 `al` 中的一个字符                                   |
| `DISPRF`  | 显示标志位寄存器，实验里常用于核对标志                   |
| `DISPHD` / `dispsid` | 显示寄存器/数据的十六进制或有符号十进制结果，按题目库函数说明使用 |
| `DISPHB`  | 显示一个字节的十六进制形式                               |
| `DISPMEM` | 从低地址到高地址逐字节显示某段内存                       |

- `.intel_syntax noprefix`
- `.data` 如需变量/字符串
- `.text`
- `.globl main`
- `main:`
- 子程序标号
- `ret`
- 除法题最好写除数为 0 分支

### 12.2 显示 EAX 32 位二进制：把 EAX 里的 32 位整数，按二进制形式一位一位显示出来。

核心思路：从最高位开始移出，CF=0 输出 `'0'`，CF=1 输出 `'1'`。

```asm
dispeax:
    push rdx
    push rcx
    mov edx, eax
    mov ecx, 32
.again:
    shl edx, 1
    mov al, '0'
    jnc .disp
    mov al, '1'
.disp:
    call dispc
    loop .again
    pop rcx
    pop rdx
    ret
```

注意：不要直接边移 `eax` 边把字符放 `al`，否则会破坏待显示的数据。

### 12.3 显示标志位：把 CPU 当前的几个状态标志位读出来，如果某个标志位为 1，就把对应字母写进字符串 str，最后调用 dispmsg 显示。

用 `pushfq/pop rax` 取标志位，然后 `test` 掩码。

常用位：

| 位  | 标志 | 掩码    |
| --- | ---- | ------- |
| D0  | CF   | `0x1`   |
| D2  | PF   | `0x4`   |
| D4  | AF   | `0x10`  |
| D6  | ZF   | `0x40`  |
| D7  | SF   | `0x80`  |
| D11 | OF   | `0x800` |

模板：

```asm
dprflags:
    pushfq #pushfq 是 把 RFLAGS 标志寄存器压入栈中。
    pop rax
    xor ebx, ebx

    test rax, 1
    jz .cf0
    mov byte ptr str[rip+rbx], 'C'
    inc rbx
.cf0:
    test rax, 0x40
    jz .zf0
    mov byte ptr str[rip+rbx], 'Z'
    inc rbx
.zf0:
    # 继续测 SF/OF/AF/PF
    mov byte ptr str[rip+rbx], 0
    lea rax, str[rip]
    call dispmsg
    ret
```

如果题目要求标志为 0 输出点 `.`，就每个标志都写一个字符，不要跳过。

### 12.4 x86-64 子程序模板

```asm
add:
    # Linux x86-64: edi = a, esi = b
    mov eax, edi
    add eax, esi
    ret
```

Win64 题面则通常把前两个整数参数放在 `ecx/edx`，写题时先看平台。

### 12.5 宏汇编题

```asm
.macro LOGICAL op, a, b

    \op eax, \b

.macro LOGICAL_NOT a

.endm

.text

    LOGICAL AND, 0xFF, 0x0F
```

### 12.6 平均值/数组输入输出类题

常见套路：

1. `count` 设循环次数。
2. 指针寄存器指向数组首地址。
3. 循环中调用输入子程序，把返回值存入数组。
4. 求和注意有符号扩展和溢出。
5. 除以前检查除数。
6. 商用于平均值输出。

### 12.7 实验型编程套路

这些来自实验/课件，未必原题照搬，但可作为综合编程题素材：

- 斐波那契：保存前两项 `a,b`，循环算 `next=a+b`；用 CF/OF 或比较判断是否超出 32 位范围；输出后更新 `a=b, b=next`。
- 冒泡排序：两层循环；外层控制趟数，内层比较相邻元素；若 `A[j] > A[j+1]` 则交换；注意数组元素大小决定地址步长。
- 素数判断：`N<=3` 直接素数；大于 3 的偶数不是素数；从 3 开始只试除奇数，余数 `edx==0` 表示能整除。
- Eratosthenes 筛法：建字节数组标记非素数；从 2 开始，把每个素数的倍数标 1；最后统计未标记的位置。
- 字符/ASCII（American Standard Code for Information Interchange，美国信息交换标准代码）表：高 4 位决定行，低 4 位决定列；二进制转十六进制字符常用查表 `"0123456789ABCDEF"`。

## 13. 临考高频错题订正

1. `ret` 在 64 位下让 `RSP += 8`，不是减 8。
2. `loop` 在 `ECX=0` 时不是 0 次，而是减到 `0xFFFFFFFF` 后跳。
3. `mov [mem], [mem]` 错，要用寄存器中转。
4. `mov [rbx], 1` 类型不明确，要 `byte/word/dword/qword ptr`。
5. `cmp` 不改两个操作数，只改标志。
6. `lea` 不访问内存，只算地址表达式。
7. `mov eax,0x6735` 后 `AL=0x35`，不是 `0x67`。
8. `and/or/xor` 后 `CF=OF=0`；`not` 不影响标志。
9. `PF=1` 表示低 8 位中 1 的个数为偶数。
10. 有符号比较用 `jg/jl`，无符号比较用 `ja/jb`。
11. 64 位模式下访问内存基址/变址用 64 位寄存器。
12. `AH/BH/CH/DH` 不能和 `SIL/DIL/BPL/SPL/R8B...` 在同一条需要 REX 的指令里混用。
13. `.asciz` 有结尾 0，`.ascii` 没有。
14. `.equ` 不分配内存。
15. `xchg` 不能有立即数，且不能两个普通内存操作数。
16. 默认写 GAS Intel x86-64，不要混入其他汇编器语法。
